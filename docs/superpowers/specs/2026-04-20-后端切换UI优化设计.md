# 后端切换 UI 优化设计

## 背景

当前后端切换机制存在以下问题：
1. 当前选中后端指示不突出，两个切换按钮无区分
2. 切换成功后无用户反馈
3. 未验证切换是否真正成功

## 设计目标

1. 明确显示当前选中的后端
2. 切换需用户确认，防止误操作
3. 切换后验证状态，反馈真实结果
4. 气泡提示切换结果

## UI 改动

### 布局调整

将"后端状态"改为"后端选择"，移除原有"当前后端"文字区，改为单选按钮组形式。

```
旧布局：
┌─────────────────────────────┐
│ 后端状态                      │
│ OpenClaw    ● 运行中         │
│ Hermes      ○ 未运行         │
│ 当前后端: OpenClaw            │
│ [切换到OpenClaw] [切换到Hermes]│
└─────────────────────────────┘

新布局：
┌─────────────────────────────┐
│ 后端选择                      │
│                             │
│ ◉ OpenClaw        ● 运行中   │
│ ○ Hermes          ● 运行中   │
│                             │
└─────────────────────────────┘
```

### 单选按钮样式

- 未选中：空心圆 ○，灰色文字
- 选中：实心圆 ●，橙色文字，左侧橙色竖线指示
- 状态指示器跟在名称后面

### 确认对话框

点击选中后立即弹出确认框：

```
┌─────────────────────────────┐
│       确认切换后端？          │
│                             │
│  切换后消息将发送至 Hermes。  │
│                             │
│     [取消]      [确认切换]   │
└─────────────────────────────┘
```

- 模态遮罩：rgba(0,0,0,0.3)
- 对话框：圆角 20px，橙色边框，奶白色背景
- 按钮：取消(灰色背景) / 确认(橙色渐变背景)

### 气泡提示

位置：右下角，距边缘 20px

**成功：**
```
┌─────────────────────┐
│ ✓ 已切换到 Hermes   │  (绿色背景 #d4edda)
└─────────────────────┘
```

**失败：**
```
┌─────────────────────────┐
│ ✗ 切换失败：Hermes 未响应 │  (红色背景 #f8d7da)
└─────────────────────────┘
```

- 2秒后自动消失
- 出现动画：从右侧滑入

## 交互流程

```
1. 用户点击 Hermes 单选按钮行
       ↓
2. 弹出确认对话框
       ↓
3. 用户点击"确认切换"
       ↓
4. 执行 switch_backend RPC
       ↓
5. 验证：调用 get_current_backend 确认返回 'hermes'
       ↓
6. 验证：调用 check_backend_health 确认 Hermes 可用
       ↓
7a. 验证通过 → 气泡"已切换到 Hermes ✓"
7b. get_current_backend 返回非 hermes → 气泡"切换失败：状态异常"
7c. Hermes 不可用 → 气泡"切换失败：Hermes 未响应"
```

## 实现细节

### 文件改动

- `src/options.ts` — 重构后端选择 UI，添加确认对话框和气泡提示

### 状态验证逻辑

```typescript
async function verifySwitch(targetBackend: string): Promise<{success: boolean, message: string}> {
  // 验证1：检查当前后端是否切换
  const current = await invoke<string | null>('get_current_backend');
  if (current !== targetBackend) {
    return { success: false, message: `切换失败：状态异常` };
  }

  // 验证2：检查目标后端是否可用
  const health = await invoke<BackendHealth>('check_backend_health');
  if (!health.available) {
    return { success: false, message: `切换失败：${targetBackend === 'hermes' ? 'Hermes' : 'OpenClaw'} 未响应` };
  }

  return { success: true, message: `已切换到 ${targetBackend === 'hermes' ? 'Hermes' : 'OpenClaw'}` };
}
```
