# Working 气泡文字省略

## 问题

Working 状态下右上角工具提示气泡显示文字时，如果文字过长会超出气泡窗口边界被截断。原因是 `.bubble-text` 设置了 `white-space: nowrap` 保持单行，但没有配合 `text-overflow: ellipsis` 和宽度约束。

## 解决方案

修改 `src/components/DesktopPet.vue` 中 `.bubble-overlay` 和 `.bubble-text` 的 CSS 样式：

### 1. `.bubble-overlay` 新增 `overflow: hidden`

确保子元素溢出时被裁剪，而不是继续占用空间。

### 2. `.bubble-text` 修改

- 添加 `max-width: 100%` — 让宽度跟随父元素 `.bubble-overlay` 的 `var(--working-bubble-width)`，而不是被文字内容撑开
- 添加 `text-overflow: ellipsis` — 文字超长时显示 `...`
- 保留 `white-space: nowrap` — 保持单行省略
- 添加 `overflow: hidden` — 确保截断生效

### 3. 动态缩放兼容性

`.bubble-overlay` 宽度通过 CSS 变量 `var(--working-bubble-width)` 动态设置，值来自 `scaled.workingBubbleWidth = BASE_LAYOUT.workingBubbleWidth * scale`。因此 `max-width: 100%` 会自动跟随窗口缩放，无需额外适配。

## 修改文件

`src/components/DesktopPet.vue`

## 修改位置

```css
.bubble-overlay {
  /* ... 现有属性 ... */
  overflow: hidden;
}

.bubble-text {
  display: inline-block;
  max-width: 100%;           /* 新增：跟随父元素宽度 */
  text-overflow: ellipsis;   /* 新增：超长显示 ... */
  overflow: hidden;          /* 新增：配合截断 */
  white-space: nowrap;       /* 保留：保持单行 */
  /* ... 其他属性保持不变 ... */
}
```

## 验证

气泡宽度设置：
- `workingBubbleLeft = 100 * scale`
- `workingBubbleWidth = 220 * scale`
- 气泡右边缘位置 = `100 + 220 = 320 * scale`
- 窗口宽度 = 360 * scale

气泡右边缘不会超出窗口边界。
