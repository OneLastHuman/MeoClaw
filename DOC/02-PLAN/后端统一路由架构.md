# 后端统一路由架构

> 创建时间：2026/04/21
> 状态：已实现

## 背景

原实现中，前端每个调用点（输入框发送、右键菜单）都硬编码调用 `send_to_openclaw`，导致切换到 Hermes 后消息仍然发送到 OpenClaw。

## 改动内容

### 1. 后端新增统一路由命令

**文件**：`src-tauri/src/lib.rs`

新增 `send_message` 命令，根据当前后端自动路由到对应客户端：

```rust
#[tauri::command]
async fn send_message(message: String, attachments: Vec<Attachment>, app: AppHandle) -> Result<(), String> {
    let state = app.state::<AppState>();
    let manager = state.backend_manager.lock().map_err(|e| e.to_string())?;

    // 获取当前后端
    let current_backend = manager.current().unwrap_or_else(|| "openclaw".to_string());

    // 根据当前后端路由
    match current_backend.as_str() {
        "hermes" => {
            let stream = manager.send_message_to_client("hermes", &message)?;
            let app_for_task = app.clone();
            tokio::spawn(async move {
                handle_hermes_stream(app_for_task, stream).await;
            });
        }
        _ => {
            // OpenClaw 使用原有的 WebSocket 方式
            let client = app.state::<Arc<OpenClawClient>>();
            client.sessions_send(&message, attachments).await?;
        }
    }

    log::info!("[Router] send_message via {}", current_backend);
    Ok(())
}
```

### 3. Hermes 附件支持扩展

Hermes 通过 HTTP API 发送，接口本身支持在 messages 中传递附件数据（未来可扩展）。

## 架构优点

1. **单一职责**：路由逻辑放在 BackendManager，符合设计原则
2. **防漏**：后端做路由保证不可能绕过
3. **一致性**：所有切换后端的入口自动生效

## 影响范围

- 输入框发送消息
- 右键菜单发送
- 右键菜单 `/new`、`/stop`、`/status` 命令
