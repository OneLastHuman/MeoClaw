# 拖动时窗口中心跟随鼠标（2026-04-18）

## 这次调整解决什么问题

之前拖动桌宠时，窗口位置是按“按下瞬间的左上角偏移”在移动。

结果是：

- 鼠标抓住猫身体中间开始拖
- 窗口仍然按原本左上角锚点跟随
- 视觉上会感觉猫没有真正被“拎住中心”

这次改成：

- 拖动过程中，窗口中心点始终对齐鼠标位置

---

## 实现方式

文件：

- `src/components/DesktopPet.vue`

在开始拖动时记录：

- 当前窗口缩放 `scaleFactor`
- 当前窗口宽高的一半 `windowHalfWidth / windowHalfHeight`

在 `pointermove` 中直接计算目标位置：

```ts
targetX = screenX * scaleFactor - windowHalfWidth
targetY = screenY * scaleFactor - windowHalfHeight
```

然后通过：

```ts
getCurrentWindow().setPosition(new PhysicalPosition(targetX, targetY))
```

持续更新窗口位置。

---

## 当前效果

拖动猫本体时：

- 鼠标会位于桌宠窗口正中心
- 不再保留按下时的左上角偏移
- 慢速拖动仍然保留原有起拖阈值

---

## 验证方式

1. 启动应用：

```bash
npm run tauri dev
```

2. 鼠标按住猫本体中间开始拖动。

3. 验证：

- 鼠标始终落在窗口中心附近
- 拖动过程中没有明显跳回左上角锚点
- 双击切换输入框行为不受影响
