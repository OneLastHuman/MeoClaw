# Hermes 自动发现机制设计方案

> 创建时间：2026/04/22
> 状态：已确认，待实现

---

## 一、背景

MeoClaw 是一个跨平台（Windows、Mac、Linux）的桌面应用，需要连接 Hermes 后端处理消息。现在的连接方式只能连本机的 Hermes，换电脑或连其他机器的 Hermes 就找不到了。

**目标：让 MeoClaw 打开后，自动找到局域网里正在运行的 Hermes 后端，用户无需任何配置。**

---

## 二、现有架构问题

| 问题 | 说明 |
|------|------|
| 只连本机 | Layer 1-3 都只看 localhost，找不到其他机器上的 Hermes |
| 硬编码端口 | 写死了 127.0.0.1:8642，discovery 发现的地址没用上 |
| 无法重连 | Hermes 挂了不会自动重试 |

---

## 三、解决方案：4 层自动发现

### 发现流程图

```
启动 MeoClaw
    ↓
┌─────────────────────────────────────────┐
│ Layer 1: 本机配置文件                     │
│ 读取 ~/.hermes/.env                      │
│ 获取用户配置的端口和地址                   │
└─────────────────────────────────────────┘
    ↓ 找不到
┌─────────────────────────────────────────┐
│ Layer 2: 本机进程扫描                     │
│ 检查 Hermes 进程是否在本机运行             │
│ 如果在运行 → 复用 Layer 1 的配置           │
└─────────────────────────────────────────┘
    ↓ 找不到
┌─────────────────────────────────────────┐
│ Layer 3: 本机端口探测                     │
│ 尝试本机 8642、9119 端口                  │
│ 用 /health 端点验证是否是 Hermes          │
└─────────────────────────────────────────┘
    ↓ 找不到（或者需要连其他机器）
┌─────────────────────────────────────────┐
│ Layer 4: 局域网安全发现  ← 新增           │
│ 读取本机 ARP 缓存（已通信过的设备）         │
│ 对每个 IP 探测常见端口                    │
│ 用 /health 端点验证是否是 Hermes          │
└─────────────────────────────────────────┘
```

---

## 四、Layer 4 详细设计

### 4.1 为什么用 ARP 缓存？

| 方式 | 安全性 | 说明 |
|------|--------|------|
| 发送 ARP 请求 | ❌ 危险，会触发杀毒软件 | 主动发广播包 |
| 读取 ARP 缓存 | ✅ 安全，只读内核数据 | 不发任何包 |

**读取 ARP 缓存**是安全的——它只是读取本机内核已经维护的 ARP 表，不发送任何数据包。

### 4.2 探测端口列表

| 端口 | 原因 |
|------|------|
| 8642 | Hermes API 默认端口 |
| 9119 | Hermes Dashboard 端口 |
| 8080 | 常见 HTTP 替代端口 |
| 8000 | 常见 HTTP 替代端口 |
| 3000 | 常见 HTTP 替代端口 |

### 4.3 验证机制

对每个可疑 IP:Port 发送 HTTP GET `/health` 请求：
- 返回正确响应 → 确认是 Hermes
- 连接失败或响应不对 → 继续下一个

## 五、多 Hermes 实例支持

### 发现多个怎么办？

MeoClaw 会把所有发现的 Hermes 都记录下来。

| 情况 | 处理方式 |
|------|---------|
| 只发现 1 个 | 自动连接 |
| 发现 2 个以上 | 全部显示在「选项 → 后端」里，用户自己选 |

### 区分不同实例

用 `IP:Port` 组合来区分。比如：
- `192.168.1.100:8642`（用户 A 的 Hermes）
- `192.168.1.101:8642`（用户 B 的 Hermes）
- `192.168.1.100:8643`（用户 A 的第二个 Hermes Profile）

---

## 六、记住用户选择

用户选了一个 Hermes 后，MeoClaw 会记住这个地址。

下次启动时：
1. 直接尝试连接上次选择的地址
2. 同时后台悄悄重新扫描一遍
3. 如果上次的选择连不上了，自动切换到新的可用 Hermes

## 八、启动时序图

```
MeoClaw 启动
    ↓
Layer 1: 读本机 .env → 找到了？→ 直接连
    ↓ 没有
Layer 2: 扫本机进程 → 找到了？→ 直接连
    ↓ 没有
Layer 3: 试本机端口 → 找到了？→ 直接连
    ↓ 没有
Layer 4: 读 ARP 缓存 → 获取 IP 列表
    ↓
对每个 IP，尝试连接 8642/9119/8080/8000/3000
    ↓
用 /health 验证
    ↓
找到了 1 个 → 直接连
找到了 2+ 个 → 显示在 UI 供用户选
一个都没找到 → 继续用之前的配置尝试
```

## 九、速度预估

| 步骤 | 时间 |
|------|------|
| Layer 1-3 | < 100ms |
| Layer 4（ARP 缓存 10 个 IP） | 1-3 秒 |
| Layer 4（ARP 缓存 50 个 IP） | 5-10 秒 |

全部在后台静默执行，用户无感知。
