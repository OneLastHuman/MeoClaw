# 分身功能设计

## 1. 概述

允许用户在桌面上创建多个宠物实例（分身），每个分身独立连接 AI session，并行处理用户任务。

- **限制**：最多 5 个分身
- **区分方式**：使用不同的精灵图
- **窗口形式**：每个分身是独立的 Tauri WebviewWindow
- **Session 隔离**：每个分身有独立的 AI session，互不干扰

## 2. 核心方案

**OpenClaw 分身共享连接，用不同 session_key 路由；Hermes 分身需独立实例。**

### 2.1 关键发现

1. **OpenClaw Gateway** 本身支持多 session，每个 session 有唯一的 session_key
2. **Hermes HTTP** 虽是无状态的，但多个请求可能被 AI 后端当作同一对话的延续，需要每个分身独立实例
3. **OpenClaw 分身**：无需为每个分身创建独立的 BackendClient 实例
4. **Hermes 分身**：需要为每个分身创建独立的 HermesClient 实例

### 2.2 架构优势

- OpenClaw 分身复用现有客户端实例，不增加连接数
- BackendManager 维护 `window_label -> session_key` 映射和 `window_label -> HermesClient` 映射
- 消息路由时根据 window_label 查找对应 session_key 和客户端实例
- 窗口关闭时只释放映射，session 不关闭（可复用）

## 3. 技术架构

### 3.0 BackendClient Trait 修改

为支持 session_key 路由，需扩展 BackendClient trait 的 `send_message` 方法签名：

```rust
pub trait BackendClient: Send + Sync {
    fn name(&self) -> &str;
    fn check_health(&self) -> Box<dyn Future<Output = Result<HealthStatus, BackendError>> + Send + '_>;
    fn send_message(
        &self,
        text: &str,
        session_key: Option<&str>,
    ) -> Result<Box<dyn Stream<Item = Result<String, BackendError>> + Send + 'static>, BackendError>;
    fn endpoint(&self) -> &str;
    fn auth_headers(&self) -> HashMap<String, String>;
}
```

**变更点**：
- `send_message` 新增 `session_key: Option<&str>` 参数
- OpenClawClientAdapter 和 HermesClient 都要实现此签名
- 调用方根据 window_label 查找对应 session_key，传给 send_message

### 3.0.1 Hermes 路由说明

Hermes 本身是无状态的，但多个 HTTP 请求可能被 AI 后端当作同一对话的延续。

**结论**：需要为每个分身创建**独立的 HermesClient 实例**，而非共享实例。

原因：
- 如果共享一个 HermesClient，多个分身的请求会混合在一起，被 AI 误认为是同一对话的继续
- 每个 HermesClient 维护独立的连接和请求上下文

**更新后的 BackendManager 结构**：

```rust
struct BackendManager {
    openclaw_client: OpenClawClient,           // 共享实例（OpenClaw 支持多 session）
    hermes_clients: HashMap<String, HermesClient>,  // window_label -> HermesClient
    clone_sessions: HashMap<String, String>,   // window_label -> session_key
    next_session_key: u32,
}
```

**session 路由逻辑**：

```rust
fn send_message(&self, window_label: &str, text: &str) -> Result<...> {
    if window_label == "main" {
        // 主宠使用默认 session
        self.openclaw_client.send_message(text, None)
    } else if let Some(session_key) = self.clone_sessions.get(window_label) {
        // 分身：根据 backend 类型选择客户端
        match self.backends.get("hermes") {
            Some(Backend::Hermes(client)) => {
                // 查找该 window_label 对应的 HermesClient 实例
                let client = self.hermes_clients.get(window_label)?;
                client.send_message(text, Some(session_key))
            }
            _ => {
                // OpenClaw 分身
                self.openclaw_client.send_message(text, Some(session_key))
            }
        }
    } else {
        Err(BackendError::WindowNotFound(window_label.to_string()))
    }
}
```

### 3.0.2 主进程重启后的状态恢复

主进程重启后，`clone_sessions HashMap` 和 `hermes_clients` 都会丢失。

**处理方案**：重启后自动清理孤儿 session，不做复杂恢复。

- 重启后 BackendManager 初始化为空状态
- 旧的分身窗口会尝试 reconnect，但会失败（session 已失效）
- 前端检测到错误后，提示用户重新创建分身
- 窗口本身可以被操作系统保留，但 AI 对话已断开

**文档化限制**：设计文档需明确说明此限制，用户需在主进程重启后重新创建分身。

### 3.0.3 未知 window_label 的错误处理

send_message 时，如果 window_label 不存在于 clone_sessions，应返回错误而非 panic：

```rust
fn send_message(&self, window_label: &str, text: &str) -> Result<...> {
    // 使用 contains_key 检查，避免 panic
    if !self.clone_sessions.contains_key(window_label) {
        return Err(BackendError::WindowNotFound(window_label.to_string()));
    }

    let session_key = self.clone_sessions.get(window_label).unwrap(); // 安全：已检查存在
    // ...
}
```

或在查找时直接用 match：

```rust
let session_key = match self.clone_sessions.get(window_label) {
    Some(key) => key,
    None => return Err(BackendError::WindowNotFound(window_label.to_string())),
};
```

## 4. Session 路由机制

### 4.1 CloneSession 结构

```rust
struct CloneSession {
    window_label: String,
    session_key: String,
}
```

注意：不再需要独立的 BackendClient 实例，只需记录 window_label 和 session_key 的映射关系。

### 4.2 BackendManager 结构

```rust
struct BackendManager {
    openclaw_client: OpenClawClient,                    // 共享实例（OpenClaw 多 session）
    hermes_clients: HashMap<String, HermesClient>,      // window_label -> HermesClient（分身独立实例）
    clone_sessions: HashMap<String, String>,            // window_label -> session_key
    next_session_key: u32,                              // 下一个可用的 session_key 编号
}
```

注意：
- OpenClaw 共享一个客户端实例，用 session_key 区分不同分身
- Hermes 每个分身需要独立实例，避免请求被当作同一对话

### 4.3 消息路由流程

```
1. 前端调用 invoke('send_message', { window_label, text })
2. BackendManager 根据 window_label 在 clone_sessions 中查找 session_key
3. 根据后端类型选择客户端：
   - OpenClaw 分身：openclaw_client.send_message(text, Some(session_key))
   - Hermes 分身：hermes_clients[window_label].send_message(text, Some(session_key))
4. 后端将 session_key 加入请求头 X-Session-Key
5. OpenClaw Gateway / Hermes 根据 session_key 路由到对应 session
6. 流式响应通过事件发回前端
```

### 4.4 主宠消息路由

主宠使用默认 session（无 session_key 或 session_key = "main"）：

```rust
fn send_message(&self, window_label: &str, text: &str) -> Result<...> {
    if window_label == "main" {
        // 主宠使用默认 session
        self.openclaw_client.send_message(text, None)
    } else {
        // 分身：先检查 window_label 是否存在
        let session_key = match self.clone_sessions.get(window_label) {
            Some(key) => key,
            None => return Err(BackendError::WindowNotFound(window_label.to_string())),
        };

        // 根据后端类型路由
        match self.backends.get("hermes") {
            Some(Backend::Hermes(_)) => {
                // Hermes 分身使用独立实例
                let client = self.hermes_clients.get(window_label)
                    .ok_or_else(|| BackendError::WindowNotFound(window_label.to_string()))?;
                client.send_message(text, Some(session_key))
            }
            _ => {
                // OpenClaw 分身
                self.openclaw_client.send_message(text, Some(session_key))
            }
        }
    }
}
```

## 5. 窗口注册流程

### 5.1 创建分身序列

```
[创建分身]
用户右键"创建分身"
  → invoke('create_clone_session')
  → BackendManager 检查上限 (< 5)
  → 生成 session_key = format!("clone-{}", next_index)
  → 生成 window_label = format!("clone-{}", available_index)
  → 存储 clone_sessions[window_label] = session_key
  → 返回 { window_label, session_key }
  → 前端 new WebviewWindow(label, config)
  → 新窗口加载完成，invoke('register_window', { window_label, session_key })
  → 验证 session_key 一致性（安全检查）
```

### 5.2 简化后的架构

- CloneSession 只需存储 window_label 和 session_key
- 不需要独立的 BackendClient 实例
- 复用现有的 OpenClawClient

### 5.3 窗口关闭清理

```rust
// 监听窗口关闭事件
window.on_window_event(|event| {
    if let WindowEvent::CloseRequested { .. } = event {
        // 从 clone_sessions 移除映射
        manager.clone_sessions.remove(&window_label);

        // 如果是 Hermes 分身，也移除其客户端实例
        manager.hermes_clients.remove(&window_label);

        // session 不关闭（Gateway 端保留），只释放本地映射
    }
});
```

清理时机：
- 监听 Tauri 窗口关闭事件
- 从 clone_sessions 中删除 window_label 对应的 entry
- 从 hermes_clients 中删除对应的 HermesClient 实例（如有）
- session_key 不关闭（Gateway 端保留），只释放本地映射

### 6.2 实现代码

```rust
#[tauri::command]
fn create_clone_session(manager: State<BackendManager>) -> Result<CloneSessionResponse, String> {
    let count = manager.clone_sessions.len();
    if count >= 5 {
        return Err("已达最大分身数量 (5)".into());
    }

    let index = manager.next_available_index();
    let window_label = format!("clone-{}", index);
    let session_key = format!("clone-{}", index);

    manager.clone_sessions.insert(window_label.clone(), session_key.clone());

    Ok(CloneSessionResponse { window_label, session_key })
}

#[tauri::command]
fn register_window(window_label: String, session_key: String, manager: State<BackendManager>) -> Result<(), String> {
    // 验证 session_key 一致性
    if manager.clone_sessions.get(&window_label) != Some(&session_key) {
        return Err("session_key 不匹配".into());
    }
    Ok(())
}

#[tauri::command]
fn close_clone_session(window_label: String, manager: State<BackendManager>) -> Result<(), String> {
    manager.clone_sessions.remove(&window_label);
    Ok(())
}

#[tauri::command]
fn get_clone_count(manager: State<BackendManager>) -> usize {
    manager.clone_sessions.len()
}
```

## 10. 事件流总览

```
[创建分身]
用户右键"创建分身"
  → invoke('create_clone_session')
  → BackendManager 分配 session_key，返回 window_label
  → 前端 new WebviewWindow(label, config)
  → 新窗口加载完成，invoke('register_window', { label, session_key })
  → BackendManager 建立 window_label → session_key 映射

[发送消息]
用户输入消息，双击发送
  → invoke('send_message', { window_label: currentLabel, text })
  → BackendManager 根据 window_label 找到 session_key
  → OpenClaw 分身：openclaw_client.send_message(text, Some(session_key))
  → Hermes 分身：hermes_clients[window_label].send_message(text, Some(session_key))
  → 流式响应通过事件发回前端

[关闭分身]
用户右键"关闭分身"
  → invoke('close_clone_session', { window_label })
  → BackendManager 删除 window_label 映射和 HermesClient 实例（如有）
  → 前端调用 window.close()
```

## 11. 边界情况处理

| 情况 | 处理方式 |
|------|----------|
| 5 个分身后再点"创建分身" | 菜单项禁用或提示"已达上限" |
| 关闭主宠 | 关闭所有窗口，退出程序 |
| 分身 AI 无响应 | 独立超时，不影响其他分身 |
| 分身窗口崩溃 | 仅该分身关闭，不影响主宠和其他分身 |
| 同一分身快速创建/关闭 | session_key 保留在 Gateway，复用时直接映射 |
| 主进程重启 | clone_sessions HashMap 丢失，旧分身窗口显示"连接已断开"，用户需重新创建分身 |
| send_message 时 window_label 不存在 | 返回 `WindowNotFound` 错误，前端提示"分身已失效" |
