# 后端切换 UI 与验证反馈

> 更新日期: 2026-05-14
> 状态: 当前实现

## 概述

重构了选项窗口中的后端选择 UI，从两个切换按钮改为单选按钮组形式，增加了确认对话框和双重验证机制，提供清晰的切换成功/失败反馈。

## 问题背景

原 UI 存在以下问题：
1. 两个切换按钮一直显示，无法区分哪个是当前选中
2. 切换成功后无用户反馈
3. 未验证切换是否真正成功
4. 后端状态显示不突出

## 实现方案

### UI 改动

**新布局：**
```
┌─────────────────────────────┐
│ 后端选择                      │
│                             │
│ ◉ OpenClaw        ● 运行中   │
│ ○ Hermes          ● 运行中   │
│                             │
└─────────────────────────────┘
```

- 单选按钮组形式，当前选中项有橙色边框和实心圆指示
- 状态指示器跟在每个后端名称后面
- 点击任意选项立即弹出确认对话框

### 交互流程

```
用户点击 Hermes 单选按钮
       ↓
弹出确认对话框
┌─────────────────────────────┐
│       确认切换后端？          │
│  切换后消息将发送至 Hermes。  │
│     [取消]      [确认切换]   │
└─────────────────────────────┘
       ↓ (用户点确认)
执行 switch_backend RPC
       ↓
双重验证：
1. 检查是否与切换前状态不同
2. 检查是否成功切换到目标后端
3. 检查目标后端是否可用
       ↓
显示结果气泡 (2秒)
```

### 气泡提示类型

| 场景 | 消息 | 样式 |
|------|------|------|
| 切换成功 | ✓ 已切换到 Hermes | 绿色 #d4edda |
| 已是该后端 | Hermes 已是当前后端 | 红色 #f8d7da |
| 状态异常 | 切换失败：状态异常 | 红色 #f8d7da |
| 后端不可用 | 切换失败：Hermes 未响应 | 红色 #f8d7da |
| 执行异常 | 切换失败：执行异常 | 红色 #f8d7da |

## 代码结构

### 文件

- `src/options.ts` — 前端选项窗口，包含后端选择 UI 和所有交互逻辑

### 关键函数

| 函数 | 职责 |
|------|------|
| `updateRadioSelection(backend)` | 更新单选按钮选中状态 |
| `openConfirmModal(backend)` | 打开确认对话框 |
| `closeConfirmModal()` | 关闭确认对话框 |
| `showToast(message, isSuccess)` | 显示气泡提示，2秒后自动消失 |
| `verifySwitch(target, previous)` | 双重验证切换结果 |
| `switchBackend(backend)` | 执行切换 + 验证 + 反馈 |
| `checkBackendStatus()` | 检查所有后端健康状态（调用 `check_all_backends_health`） |

### Tauri RPC 命令

| 命令 | 职责 |
|------|------|
| `switch_backend` | 执行后端切换 |
| `get_current_backend` | 获取当前选中的后端 |
| `check_all_backends_health` | 检查所有已注册后端健康状态 |
| `check_backend_health` | 检查当前后端健康状态（仅选中后端） |

## 验证逻辑

```typescript
async function verifySwitch(targetBackend: string, previousBackend: string | null) {
  // 验证1: 检查是否与切换前不同
  if (previousBackend === targetBackend) {
    return { success: false, message: `${backendName} 已是当前后端` };
  }

  // 验证2: 检查是否成功切换到目标后端
  const current = await invoke<string | null>('get_current_backend');
  if (current !== targetBackend) {
    return { success: false, message: '切换失败：状态异常' };
  }

  // 验证3: 检查目标后端是否可用
  const healthList = await invoke<BackendHealth[]>('check_all_backends_health');
  const targetHealth = healthList.find(h => h.backend === targetBackend);
  if (!targetHealth?.available) {
    return { success: false, message: `切换失败：${backendName} 未响应` };
  }

  return { success: true, message: `已切换到 ${backendName}` };
}
```
