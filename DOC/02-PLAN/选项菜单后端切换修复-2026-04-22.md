# 选项菜单后端切换修复

## 问题描述

选项菜单中的后端切换功能完全不工作。用户点击 OpenClaw 或 Hermes 后，会弹出确认框，但点击确认后显示错误：

```
FAIL Switch Failed: Exec Error
```

但实际上 OpenClaw 和 Hermes 后端都在正常运行。

## 根本原因

问题出在 `src/options.ts` 文件中，确认对话框的回调函数存在 bug：

```typescript
// 修复前的代码
document.getElementById('confirm-ok')?.addEventListener('click', async () => {
  if (pendingBackend) {
    closeConfirmModal();  // ← 这里把 pendingBackend 设为 null！
    await switchBackend(pendingBackend);  // ← pendingBackend 现在是 null
  }
});
```

`closeConfirmModal()` 函数内部会执行 `pendingBackend = null`：

```typescript
function closeConfirmModal() {
  pendingBackend = null;  // ← 副作用
  const modal = document.getElementById('confirm-modal');
  if (modal) {
    modal.classList.add('hidden');
  }
}
```

因此当调用 `switchBackend(pendingBackend)` 时，传入的值已经是 `null`，导致 Rust 端报错：

```
invalid args `backend` for command `switch_backend`: invalid type: null, expected a string
```

## 修复方案

在调用 `closeConfirmModal()` 之前先保存 `pendingBackend` 的值：

```typescript
// 修复后的代码
document.getElementById('confirm-ok')?.addEventListener('click', async () => {
  if (pendingBackend) {
    const backendToSwitch = pendingBackend;  // 先保存值
    closeConfirmModal();  // closeConfirmModal 会把 pendingBackend 设为 null，所以先保存
    await switchBackend(backendToSwitch);
  }
});
```

## 修改文件

- `src/options.ts` - 修复了确认对话框的回调函数

## 测试验证

1. 打开选项菜单 (Options)
2. 进入 BACKEND 面板
3. 点击 Hermes 或 OpenClaw
4. 确认切换后应该显示成功提示：
   - "Switched to Hermes" 或 "Switched to OpenClaw"

## 相关信息

- 配置文件位置: `~/Library/Application Support/meo-claw/backend-config.json`
- 当前实现的后端: Hermes (默认), OpenClaw
