# 桌宠贴边功能实现（2026-04-21）

## 概述

贴边功能允许桌宠在拖动到屏幕边缘时吸附隐藏，只露出部分身体。系统支持两种贴边模式：

- **吸附隐藏（Snap）**：拖动松手后直接吸附到边缘，播放入口动画
- **Peek 悬停**：鼠标进入隐藏区域后猫咪探出，播放 hover 动画

---

## 状态机

### 窗口行为状态（WindowBehaviorState）

```
┌─────────┐
│  normal │
└────┬────┘
     │ drag
     ▼
┌─────────┐
│ dragging │
└────┬────┘
     │ release in snap zone
     ▼
┌──────────────────┐    mouse enters    ┌──────────────────┐
│  edgeHiddenLeft  │◄─────────────────►│   edgePeekLeft   │
└──────────────────┘    mouse leaves   └──────────────────┘
     │                                       │
     │ snap to right                         │ snap to right
     ▼                                       ▼
┌──────────────────┐                   ┌──────────────────┐
│ edgeHiddenRight  │◄─────────────────│  edgePeekRight   │
└──────────────────┘                   └──────────────────┘
```

### 状态说明

| 状态 | 说明 |
|------|------|
| `normal` | 普通状态，可在桌面任意位置 |
| `dragging` | 拖动中 |
| `edgeHiddenLeft` | 吸附隐藏于左侧，只露出 65% |
| `edgeHiddenRight` | 吸附隐藏于右侧，只露出 65% |
| `edgePeekLeft` | 左侧 peek 探出，露 50%，hover 动画循环 |
| `edgePeekRight` | 右侧 peek 探出，露 50%，hover 动画循环 |

### 双状态系统

桌宠存在两套独立的状态系统：

1. **AnimState**（动画状态）：控制猫咪本身的动画帧播放
   - `idle`、`EnterInput`、`startworking`、`working`、`Receiving`、`Response` 等

2. **WindowBehaviorState**（窗口行为状态）：控制窗口位置和贴边行为
   - `normal`、`dragging`、`edgeHiddenLeft/Right`、`edgePeekLeft/Right`

两者正交：猫咪可以在任何窗口行为状态下播放任何动画。

## 行为参数

| 参数 | 值 | 说明 |
|------|-----|------|
| `EDGE_HIDDEN_REVEAL_RATIO` | `0.65` | 隐藏时露出比例（65%） |
| `EDGE_PEEK_REVEAL_RATIO` | `0.5` | peek 探出时露出比例（50%） |
| `EDGE_PEEK_BUFFER_RATIO` | `0.25` | peek 热区缓冲（窗口宽 25%） |
| `EDGE_PEEK_LEAVE_DELAY` | `0` | peek 离开延迟（ms） |
| `getEdgeSnapThreshold()` | `windowWidth` | 吸附阈值 = 猫窗口宽度 |

### 吸附判定

拖动松手时，根据鼠标屏幕坐标判断吸附方向：

```typescript
const leftGap = Math.abs(cursorX - monitor.x);
const rightGap = Math.abs(monitor.x + monitor.width - cursorX);

if (leftGap <= threshold && leftGap <= rightGap) return 'left';
if (rightGap <= threshold) return 'right';
return null;
```

### 目标位置计算

```typescript
case 'edgeHiddenLeft':
  return monitor.x - windowWidth + hiddenRevealPx; // 左侧边缘 - 窗口宽 + 露出量
case 'edgeHiddenRight':
  return monitor.x + monitor.width - hiddenRevealPx; // 右侧边缘 - 露出量
case 'edgePeekLeft':
  return monitor.x - windowWidth + peekRevealPx; // 探出时多露一点
case 'edgePeekRight':
  return monitor.x + monitor.width - peekRevealPx;
```

## 动画系统

### 素材规格

| 动画 | 文件 | 尺寸 | 排列 | 帧数 | 单帧尺寸 |
|------|------|------|------|------|----------|
| 入口动画 | `edgeEnter.png` | 3840×3840 | 4×4 | 13 | 960×960 |
| Hover 循环 | `edgePeekLoop.png` | 2880×1920 | 3×2 | 6 | 960×960 |
| 隐藏静态 | `hide_right.png` | 720×180 | 4×1 | 4 | 180×180 |
| 拖动动画 | `dragging.png` | 900×180 | 5×1 | 5 | 180×180 |

素材路径：`public/anim/edge-placeholders/`

### edgeEnter 动画（吸附入口）

- **用途**：拖动吸附成功后的入口动画
- **播放方式**：
  - 吸附进入（snap）：第13帧 → 第1帧（倒序，~0.3秒）
  - peek 进入：第1帧 → 第13帧（正序，~0.3秒）
  - peek 离开：第13帧 → 第1帧（倒序，~0.3秒）
- **每帧间隔**：23ms
- **镜像**：左侧使用 `ctx.scale(-1, 1)` 逐帧镜像

### edgePeekLoop 动画（hover 循环）

- **用途**：鼠标进入隐藏区域后的 hover 循环动画
- **播放方式**：第1帧 → 第6帧（正序循环，1.5秒/循环）
- **每帧间隔**：250ms
- **镜像**：左侧使用 `ctx.scale(-1, 1)` 逐帧镜像

### 精灵图渲染

Canvas 渲染方案（非 CSS background-position）：

```typescript
const frameSize = 960;
const srcX = (frame % cols) * frameSize;
const srcY = Math.floor(frame / cols) * frameSize;

// 右侧：正常绘制
ctx.drawImage(image, srcX, srcY, frameSize, frameSize, 0, 0, displaySize, displaySize);

// 左侧：先镜像再绘制
if (direction === 'left') {
  ctx.save();
  ctx.translate(displaySize, 0);
  ctx.scale(-1, 1);
  ctx.drawImage(image, srcX, srcY, frameSize, frameSize, 0, 0, displaySize, displaySize);
  ctx.restore();
}
```

### 精灵图隐藏机制

Canvas 动画播放期间，背景精灵图使用 `opacity: 0` 隐藏：

```css
.is-edge-enter-hidden {
  opacity: 0;
}
```

这样做的原因：
- `display: none` 会阻止 pointer events，导致无法拖动
- `opacity: 0` 保持精灵图可交互，但视觉上隐藏

## 行为流程

### 吸附流程（Snap）

```
用户拖动猫咪 → 松手在边缘吸附区
    │
    ▼
计算吸附方向（left/right）
    │
    ▼
瞬间移动到贴边位置（65% 露出）
    │
    ▼
切换到 edgeHiddenLeft/Right 状态
    │
    ▼
播放 edgeEnter 倒序动画（13→1帧，阻塞交互）
    │
    ▼
动画结束，停在第1帧，可正常交互
```

### Peek 入口流程

```
edgeHidden → 鼠标进入 peek 热区
    │
    ▼
切换到 edgePeekLeft/Right 状态
    │
    ▼
播放 edgeEnter 正序动画（1→13帧，~0.3秒，不阻塞交互）
    │
    ▼
播放 edgePeekLoop 循环动画（1.5秒/循环）
    │
    ▼
鼠标离开 → 取消 loop → 播放 edgeEnter 倒序（13→1帧）→ 回到 edgeHidden
```

### 拖动交互

**任何贴边状态下都可以直接拖动**：

1. mousedown 在精灵图上
2. 立即切换到 `dragging` 状态
3. 取消所有 edge 动画（edgeEnter、edgePeekLoop）
4. 播放 `dragging.png` 动画
5. 拖动过程中实时更新窗口位置
6. 松手后按位置判定是否吸附

## 实现细节

### 核心状态变量

```typescript
// edgeEnter 动画控制
const isEdgeEnterPlaying = ref(false);
const edgeEnterFrame = ref(0);
const edgeEnterDirection = ref<'left' | 'right'>('right');
const edgeEnterMode = ref<'snap' | 'peek'>('snap');
let edgeEnterAnimationTimer: number | null = null;

// edgePeekLoop 动画控制
const isEdgePeekLoopPlaying = ref(false);
const edgePeekLoopFrame = ref(0);
const edgePeekLoopDirection = ref<'left' | 'right'>('right');
let edgePeekLoopTimer: number | null = null;
```

### Canvas 元素

edgeEnter 和 edgePeekLoop 共用同一个 Canvas 元素：

```vue
<canvas
  ref="edgeEnterCanvasRef"
  class="edge-enter-canvas"
  :class="{ 'is-edge-enter-hidden': isEdgeEnterPlaying || isEdgePeekLoopPlaying }"
/>
```

Canvas 尺寸使用 CSS `--window-width` / `--window-height` 变量，与精灵图渲染尺寸解耦。

### 帧计算

```typescript
const totalCols = 4;
const totalRows = 4;
const frameSize = 3840 / 4; // 960

function getFramePosition(frame: number) {
  return {
    srcX: (frame % totalCols) * frameSize,
    srcY: Math.floor(frame / totalCols) * frameSize,
  };
}
```

### 镜像绘制

左侧动画通过 `ctx.scale(-1, 1)` 实现逐帧镜像（不是整图镜像）：

```typescript
function drawMirroredFrame(ctx: CanvasRenderingContext2D, image: HTMLImageElement, frame: number) {
  const { srcX, srcY } = getFramePosition(frame);

  ctx.save();
  ctx.translate(displaySize, 0);  // 移动到右侧
  ctx.scale(-1, 1);               // 水平镜像
  ctx.drawImage(image, srcX, srcY, frameSize, frameSize, 0, 0, displaySize, displaySize);
  ctx.restore();
}
```

### 动画时序

```
edgeEnter 正序 (snap=false):
  frame 0 → 12, 23ms/帧, 共 276ms

edgeEnter 倒序:
  frame 12 → 0, 23ms/帧, 共 276ms

edgePeekLoop 循环:
  frame 0 → 5 → 0 (pingpong), 250ms/帧, 1.5s/循环
```

### AnimationConfig 接口

```typescript
export interface AnimationConfig {
  name: string;           // 动画名称
  cols: number;           // 精灵图列数
  rows: number;          // 精灵图行数
  totalFrames: number;   // 总帧数
  interval: number;      // 每帧间隔(ms)
  loop: boolean;         // 是否循环

  startFrame: number;    // 起始帧索引
  imagePath: string;     // 精灵图路径
  nextState?: AnimState; // 非循环动画播放完成后的目标状态
}
```
