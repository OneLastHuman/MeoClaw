# Hermes 工具调用事件丢失问题

**日期**: 2026-04-24
**状态**: 已修复 ✅
**影响版本**: commit 0c7dee3 起

---

## 1. 问题描述

Hermes 后端发送消息时，工具调用（tool call）的进度事件没有正常返回对应的气泡消息。

**预期行为**：
- 当 AI 调用工具（如 `read_file`、`bash` 等）时，桌宠应显示工具调用的气泡
- 气泡应显示工具名称和执行状态（如 "running"）

**实际行为**：
- 消息发送后只显示 "思考中..." bubble
- 工具调用时没有任何气泡反馈
- 直接等待最终响应

---

## 2. 根本原因

**commit 0c7dee3** (`feat(Hermes): 实现自动发现机制`) 重构 SSE 解析逻辑时，**误删了 `hermes.tool.progress` 事件的处理代码**。

### 被删除的代码 (lib.rs)

```rust
// Handle hermes.tool_progress event
if current_event == "hermes.tool.progress" {
    if let Ok(json) = serde_json::from_str::<serde_json::Value>(data) {
        let tool = json["tool"].as_str().unwrap_or("unknown");
        let emoji = json["emoji"].as_str().unwrap_or("");
        let label = json["label"].as_str().unwrap_or("");
        let detail = if emoji.is_empty() {
            label.to_string()
        } else {
            format!("{} {}", emoji, label)
        };

        let payload = BubblePayload {
            tools: vec![ToolCall {
                name: tool.to_string(),
                status: "running".to_string(),
                detail,
            }],
        };
        let _ = app_for_task.emit("update-bubble", payload);
    }
    current_event.clear();
    continue;
}
```

### 当前代码的问题

1. 移除了 `current_event` 变量来追踪 event 类型
2. 移除了 `hermes.tool.progress` 事件的处理分支
3. SSE 解析只处理 `data: ` 开头的行，忽略了 `event: xxx` 行

---

## 3. SSE 事件格式

Hermes 返回的 tool progress 事件格式：

```
event: hermes.tool.progress
data: {"tool":"read_file","emoji":"📖","label":"Reading file: config.json"}
```

当前代码会：
1. 遇到 `event: hermes.tool.progress` → 直接 `continue` 跳过
2. 遇到 `data: {...}` → 尝试解析为 chat completion delta，忽略 tool progress 结构

---

## 4. 修复方案

### 方案 A：恢复 event 追踪逻辑

在 `lib.rs` 的 `send_message` Hermes 分支中，恢复 `current_event` 变量和 `hermes.tool.progress` 处理分支：

```rust
let mut current_event = String::new();
for line in text.lines() {
    let line = line.trim();
    if line.is_empty() { continue; }

    // 处理 event: xxx 行
    if let Some(event_name) = line.strip_prefix("event: ") {
        current_event = event_name.to_string();
        continue;
    }

    // 处理 data: xxx 行
    if let Some(data) = line.strip_prefix("data: ") {
        // 恢复 hermes.tool.progress 处理
        if current_event == "hermes.tool.progress" {
            // ... 原有逻辑
        }
        // ... 其他处理
        current_event.clear();
    }
}
```

### 方案 B：分离事件类型处理

将 SSE 事件处理抽象为独立函数，正确路由不同事件类型。
