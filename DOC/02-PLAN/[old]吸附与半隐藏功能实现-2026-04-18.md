## 状态流调整

当前窗口行为状态流变成：

```text
normal -> dragging -> edgeHiddenLeft
                   -> edgeHiddenRight
                   -> normal
```

其中：

- `dragging` 继续显示 `dragging.png`
- `edgeHiddenLeft` 显示 `hide_left.png`
- `edgeHiddenRight` 显示 `hide_right.png`

业务动画状态 `idle / working / Response ...` 仍然保留，但在半隐藏态下只作为底层业务状态存在，不再直接展示完整输入框和气泡。

## 实现细节

### 1. 吸附判定基准

拖拽结束时不再用窗口左上角判断，而是用鼠标屏幕坐标判断：

- 左侧距离：`cursorX - monitorLeft`
- 右侧距离：`monitorRight - cursorX`

阈值直接等于当前猫窗口宽度：

```text
snapThreshold = compactWindowWidth
```

这样在 1x 缩放下默认就是 `180px`，符合需求文档里的“一个身位”规则。

### 2. 半隐藏露出比例

半隐藏露出量改成：

```text
hiddenRevealPx = compactWindowWidth * 0.25
```

在 1x 下等于 `45px`。

对应目标位置：

- 左侧：`monitorLeft - windowWidth + hiddenRevealPx`
- 右侧：`monitorRight - hiddenRevealPx`

### 3. 贴边动画

拖拽松手且命中吸附区后：

1. 先切到 `edgeHiddenLeft / edgeHiddenRight`
2. 将窗口尺寸收回到猫本体紧凑尺寸
3. 用 `requestAnimationFrame` 执行 `500ms ease-in-out` 位移动画
4. 平滑滑到目标贴边位置

### 4. 半隐藏态的界面约束

为了保证隐藏态只显示贴边猫体，本次增加了这些限制：

- 隐藏态不显示输入框
- 隐藏态不显示 `working` 气泡
- 隐藏态不显示 `Response` 大气泡
- 隐藏态不显示 `received` 文件覆盖层

## 第二轮修复记录

### 1. 现象

上线初版后，实际拖拽验证中发现了 3 个问题：

- 猫咪会偶发跳回屏幕中间附近
- 吸附过程中没有正确播放 `hide_left.png / hide_right.png`
- 半隐藏停住后，露出区域看起来像“整张图被挤扁”，不是按帧裁切后的动画

### 2. 错误原因

#### 窗口位置跳变

初版对窗口尺寸变化使用的是理想化写法：

```text
setSize(newSize)
setPosition(oldX, oldBottom - newHeight)
```

这在逻辑上看似成立，但 macOS 实际会在 `setSize()` 后按自己的窗口扩缩行为先改变窗口位置。

如果不在 `setSize()` 之后重新读取真实位置再补偿，就可能出现：

- 锚点被破坏
- 多次同步逻辑互相覆盖
- 窗口看起来突然跳到不对的位置

#### 精灵图被当成单帧

`hide_left.png` 和 `hide_right.png` 实际文件尺寸是：

```text
720 x 180
```

也就是：

- 每帧 `180 x 180`
- 共 `4` 帧

但初版配置误写成了：

- `cols = 1`
- `rows = 1`
- `totalFrames = 1`

结果就是整张 `720x180` 精灵图被当成一帧背景图塞进 `180x180` 容器里，所以视觉上像被压缩成一张。

### 3. 修正方案

#### 窗口缩放改成“先缩放，再读回，再补偿”

这轮新增了统一的窗口锚点修正流程：

1. 先记录旧窗口的左下锚点
2. 调用 `setSize()`
3. 重新读取 `outerPosition()`
4. 计算实际偏移量
5. 再调用 `setPosition()` 做补偿

这样窗口无论进入半隐藏、退出半隐藏，还是其他模式切换，都尽量以左下角为稳定锚点。

#### 半隐藏动画按真实 4 帧接入

这轮把 `hide_left / hide_right` 改成了真实配置：

- `cols = 4`
- `rows = 1`
- `totalFrames = 4`

这样：

- 吸附过程中会播放正确帧序
- 半隐藏停驻时也不再出现整张图压缩显示

### 4. 这次得到的经验

1. 占位精灵图不能只看文件名，必须先确认实际像素尺寸和帧布局。
2. macOS 下窗口 `setSize()` 后不要直接假设位置没变，必须读回真实 `outerPosition()`。
3. 贴边功能不能只验证“有没有移动到边上”，还要验证：
   - 帧序是否正确
   - 左右方向是否正确
   - 裁切是否正确
   - 多次拖拽后位置是否稳定

### 5. 当前结论

这轮修复后，问题已处理完成：

- 偶发跳位问题已收敛
- 吸附过程已接上 `hide_left / hide_right` 动画
- 半隐藏停驻时不再把整张精灵图压成一张

当前状态可以作为后续继续实现 `edgePeek` 的稳定基础。
