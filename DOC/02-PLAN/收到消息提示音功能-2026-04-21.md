# 收到消息提示音功能

> 创建时间：2026-04-21

## 功能概述

当后端返回真正的回复消息时，自动播放选定的提示音效。用户在设置菜单中可以切换不同的音效或关闭提示音。

## 实现方案

### 触发时机

音效在后端返回最终回复时触发，而非前端状态变化：

- **OpenClaw 后端**：在 `set_response_callback` 回调触发时
- **Hermes 后端**：在流结束 `is_final: true` 且 `text_delta` 非空时

### 音效文件

路径：`public/sound/response/`

| 文件名 | 风格描述 |
|--------|---------|
| chime.mp3 | 清脆铃声 |
| soft.mp3 | 柔和提示音 |
| notification.ogg | 通知音 |
| classic.wav | 经典音效 |
| crisp.wav | 清亮短促 |

### 2. `src/components/DesktopPet.vue`

添加音效播放函数和监听逻辑：

```typescript
// 导入
import {
  responseSound,
  setResponseSound,
  getResponseSoundFile,
  RESPONSE_SOUND_OFF,
} from '../stores/petSettings';

// 播放函数
function playResponseSound() {
  const soundId = responseSound.value;
  if (soundId === RESPONSE_SOUND_OFF) return;

  const soundFile = getResponseSoundFile(soundId);
  if (!soundFile) return;

  const audio = new Audio(soundFile);
  audio.volume = 1;
  audio.play().catch(...);
}

// 监听后端回复事件
unlistenResponseBubble = await listen<ResponseBubblePayload>(
  'update-response-bubble',
  (event) => {
    if (event.payload.text) {
      responseBubbleText.value = event.payload.text;
      playResponseSound();  // 收到真正回复时播放音效
    }
  },
);
```

## 交互流程

1. 用户打开设置菜单
2. 在"收到消息提示音"区块选择音效

4. 设置自动保存到 localStorage
5. 桌宠收到后端回复时播放选定音效

## Bug 修复 (2026-04-22)

### 问题描述

使用 Hermes 后端时，收到回复消息没有播放提示音，但 OpenClaw 后端正常。

### 根本原因

`lib.rs` 中 Hermes 事件处理逻辑问题：

1. Hermes 可能先发送 `finish_reason="stop"` 再发送内容 chunks
2. 当 `is_final=true` 但 `full_text` 仍为空时，代码直接 emit 空文本然后 return
3. 后续内容 chunks 被忽略，前端收到空的 `text`

### 修复方案

移除 `else if is_final` 分支的提前 emit，让 `[DONE]` 处理时统一 emit 累积的 `full_text`。

**修改位置：** `src-tauri/src/lib.rs` 第 340-360 行

```rust
// 修复前：else if is_final 分支提前 emit 并 return
if let Some(content) = json["choices"][0]["delta"]["content"].as_str() {
    full_text.push_str(content);
} else if is_final {
    // 结束但没有 content，提前 emit（此时 full_text 可能为空）
    let _ = app_for_task.emit("update-response-bubble", serde_json::json!({ "text": full_text }));
    return;
}

// 修复后：只收集 content，让 [DONE] 统一处理最终 emit
if let Some(content) = json["choices"][0]["delta"]["content"].as_str() {
    full_text.push_str(content);
}
// 不在这里处理 finish_reason，让 [DONE] 来处理最终 emit
```

## Bug 修复 (2026-04-22) - OpenClaw 多次音效

### 问题描述

使用 OpenClaw 后端时，每条回复会触发多次提示音效（正常应该一次）。Hermes 后端无此问题。

### 根本原因

OpenClaw 客户端有多个地方调用 `response_callback`：

1. **`agent` 事件 `stream == "assistant"`**：每个 delta 都触发 callback
2. **`chat` 事件**：直接发送文本内容
3. **`session.message`**：直接发送文本内容（/status、/stop 等命令）

前端每次收到 `update-response-bubble` 都播放音效，导致多次触发。

### 修复方案

**后端 (`src-tauri/src/openclaw/client.rs`)：**

1. Delta 处理：只累积文本，不再每次 delta 都发送 callback
2. Lifecycle 处理：在 `phase == "end"` 时才发送累积的最终响应
3. Chat 事件：移除 `response_callback` 调用，只保留 `lifecycle_callback`

```rust
// 修改 1: delta 处理 - 只累积不发送
} else if agent_event.stream == "assistant" {
    if let Some(delta) = &agent_event.data.delta {
        if !delta.is_empty() {
            let mut acc = self.accumulated_response.write().await;
            acc.push_str(delta);
        }
    }
}

// 修改 2: lifecycle end 时发送累积响应
if phase == "end" {
    let callback = self.response_callback.read().await;
    if let Some(cb) = callback.as_ref() {
        let acc = self.accumulated_response.read().await;
        if !acc.is_empty() {
            cb(acc.clone());
        }
    }
    self.clear_response();
}
```

**前端 (`src/components/DesktopPet.vue`)：**

添加去重 safeguard，防止边界情况触发多次音效：

```typescript
let lastResponseText = '';
unlistenResponseBubble = await listen<ResponseBubblePayload>(
  'update-response-bubble',
  (event) => {
    if (event.payload.text && event.payload.text !== lastResponseText) {
      lastResponseText = event.payload.text;
      responseBubbleText.value = event.payload.text;
      playResponseSound();
    }
  },
);
```
