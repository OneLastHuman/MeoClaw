# 桌宠大小调整功能实现记录（2026-04-17）

## 背景

这次改动解决的是一组连在一起的问题，而不只是“加一个缩放滑块”：

- 桌宠需要支持 `50% ~ 200%` 的统一缩放
- 缩放必须同时影响主窗口、精灵图、输入框、Working 气泡、Response 气泡和文件拖拽浮层
- 状态切换时要保持猫咪左下角锚点稳定，不能因为窗口变大或缩小而“跳”
- 托盘 `选项` 菜单必须真正打开可用的设置窗口
- 选项窗口本身不能再出现内容显示不全的问题

## 最终方案

### 1. 统一缩放状态与布局计算

新增 [src/stores/petSettings.ts](/Users/hue/Documents/桌宠项目/MeoClaw/src/stores/petSettings.ts) 作为单一真相来源，集中管理：

- `petScale`
- `localStorage` 持久化
- 缩放范围、默认值、步进
- 三种窗口模式：`normal` / `working` / `response`
- 每种模式在当前缩放下的窗口尺寸
- 供 CSS 使用的缩放变量

核心思路：

- 先定义基础布局常量
- 再根据 `WindowMode + scale` 计算窗口逻辑尺寸
- 最后由组件消费同一套结果，不再各写一套常量

### 2. 主窗口按“左下角锚点”同步尺寸

[src/components/DesktopPet.vue](/Users/hue/Documents/桌宠项目/MeoClaw/src/components/DesktopPet.vue) 已重构为：

- `animState -> windowMode`
- `windowMode + petScale -> currentWindowMetrics`
- `watch([currentWindowMode, petScale]) -> syncWindowBounds()`

`syncWindowBounds()` 的处理方式：

1. 读取当前窗口 `scaleFactor / outerPosition / outerSize`
2. 将逻辑尺寸换算为物理尺寸
3. 用 `targetY = oldY + oldHeight - newHeight` 保持窗口底边不变
4. 先 `setSize()` 再 `setPosition()`

这样做的结果是：

- 缩放时猫不会上下漂
- `normal / working / response` 切换时猫也不会上下漂
- 不再依赖固定 `DPR = 2` 的写死假设

### 3. 精灵图缩放方式

这次没有用 `transform: scale()` 做整体视觉缩放，而是直接按 `petScale` 缩放：

- `backgroundPosition`
- `backgroundSize`
- 精灵容器 `width / height`

这样可以避免：

- 视觉变大但布局尺寸没变
- 窗口裁剪内容
- 动画帧错位

### 4. Working / Response / 文件浮层统一接入 CSS 变量

缩放变量已接入这些区域：

- 主精灵
- 输入框
- Working 气泡
- Response 气泡
- 文件拖拽浮层

对应文件：

- [src/components/DesktopPet.vue](/Users/hue/Documents/桌宠项目/MeoClaw/src/components/DesktopPet.vue)
- [src/components/FileDropOverlay.vue](/Users/hue/Documents/桌宠项目/MeoClaw/src/components/FileDropOverlay.vue)

### 5. `workingPreview` 状态补齐

[src/config/animations.ts](/Users/hue/Documents/桌宠项目/MeoClaw/src/config/animations.ts) 已补齐：

这解决了之前代码里已经在用、但类型系统里不存在的问题。

## 与旧方案的差异

这次实现和之前会话里“先有一个大概方向”相比，有几个关键收敛：

### 不再使用固定 `* 2`

旧思路默认 Retina 屏 `DPR = 2`，大量代码直接乘 `2`。  
现在统一使用运行时 `scaleFactor()`，避免在不同显示器或缩放环境下出错。

### 不再让 `DesktopPet.vue` 直接维护所有窗口尺寸常量

旧版本里：

- `NORMAL_WIDTH`
- `WORKING_WIDTH`
- `RESPONSE_WIDTH`
- 各类 bubble 常量

都散落在组件里。现在这些基础布局和模式计算已经收口到 `petSettings.ts`。

### 不再把 `options` 当作预留功能

托盘 `选项` 已经是可用功能，不再是设计占位。
