# Response 气泡高度自适应（2026-04-17）

## 概述

Response 状态下的气泡窗口不再固定为整块 `500px` 高度，而是根据当前内容的实际高度动态收缩或扩展。

目标有两个：

- 内容很短时，不再出现大块空白
- 内容变长时，仍然保留最大高度和滚动能力

## 当前实现

实现位置：

- `src/components/DesktopPet.vue`

核心做法：

1. 为 `.response-bubble` 增加 `ref="responseBubbleRef"`
2. 在 `Response` 状态下，通过 `offsetHeight` 读取气泡实际渲染高度
3. 用 `responseBubbleBottom + effectiveHeight` 计算当前窗口应有高度
4. 把这个高度写入 `responseWindowHeightOverride`
5. 继续复用现有 `syncWindowBounds()`，按左下角锚点重设窗口尺寸和位置

## 关键约束

- 最大高度仍然受 `responseBubbleHeight` 限制，当前基线值是 `500`
- 超出最大高度后，气泡内部继续滚动，不再继续拉高窗口
- 离开 `Response` 状态时，动态高度覆盖值会被清空，窗口回到正常模式尺寸

## 和缩放功能的关系

这个功能建立在统一缩放模型之上：

- `responseBubbleBottom`
- `responseBubbleHeight`
- `petScale`

这些值都来自 `src/stores/petSettings.ts` 的统一布局模型，所以在不同缩放倍数下，高度测量和窗口尺寸计算仍然一致。

## 验证要点

建议至少手测这几项：

1. 短文本 Response：窗口应明显变矮，不保留大片空白
2. 长文本 Response：窗口高度封顶，内部出现滚动
3. `idle -> Response -> idle`：猫脚位置保持不跳
4. 切换缩放倍数后再次进入 Response：自适应高度仍然正确
