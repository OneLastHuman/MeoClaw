# 屏幕空间感知与点击穿透实现参考

## 概述

桌面宠物应用有两个关键的系统集成问题：
1. **屏幕空间感知** — 宠物需要知道屏幕边界、任务栏高度、多显示器配置
2. **点击穿透** — 透明窗口区域点击时需要穿透到桌面，宠物实体区域才能响应点击

## 一、屏幕空间感知

### 核心问题

- 宠物不能超出屏幕边界
- 宠物不能被任务栏遮挡
- 多显示器切换时需要正确跟随
- 边角"睡觉"等特殊行为需要感知边界

### Tauri 窗口位置 API

```rust
// 获取所有显示器信息
#[tauri::command]
fn get_monitors(app: AppHandle) -> Vec<MonitorInfo> {
    app.available_monitors()
        .unwrap()
        .enumerate()
        .map(|(i, m)| MonitorInfo {
            name: m.name().unwrap_or_else(|| "Unknown".to_string()),
            x: m.position().x,
            y: m.position().y,
            width: m.size().width,
            height: m.size().height,
            is_primary: m.is_primary(),
        })
        .collect()
}

#[derive(serde::Serialize)]
struct MonitorInfo {
    name: String,
    x: i32,
    y: i32,
    width: u32,
    height: u32,
    is_primary: bool,
}
```

### 边界检测逻辑

```rust
fn ensure_on_screen(x: i32, y: i32, pet_width: u32, pet_height: u32) -> (i32, i32) {
    let monitors = app.available_monitors().unwrap();
    
    // 找到包含宠物的显示器
    for m in monitors {
        let pos = m.position();
        let size = m.size();
        
        // 检查是否完全在显示器内
        if x >= pos.x && x + pet_width as i32 <= pos.x + size.width as i32
        && y >= pos.y && y + pet_height as i32 <= pos.y + size.height as i32 {
            return (x, y);
        }
    }
    
    // 如果不在任何显示器内，回到主显示器
    let primary = monitors.find(|m| m.is_primary()).unwrap();
    (primary.position().x, primary.position().y)
}
```

## 二、点击穿透实现

### 核心问题

桌面宠物的透明背景区域不应该响应鼠标事件，只有"宠物实体"区域才应该响应。

### Tauri 原生方案

```rust
// 设置整个窗口忽略鼠标事件（点击穿透）
window.set_ignore_cursor_events(true).unwrap();
```

但这会让整个窗口不可点击，无法实现"部分穿透"。

### 高级方案：鼠标消息转发

参考 [掘金 - Tauri 点击穿透与鼠标事件转发](https://juejin.cn/post/7363101450824155163)，实现精细的点击穿透控制。

#### 实现原理

```
1. 窗口设置为透明背景（WS_EX_TRANSPARENT）
2. 通过全局鼠标钩子（WH_MOUSE_LL）捕获鼠标事件
3. 判断鼠标位置是否在"宠物实体"区域内
4. 如果在实体区域内，转发事件给 WebView
5. 如果不在，事件穿透到桌面
```

#### Rust 端实现

```rust
// 添加依赖
cargo add windows -F Win32_Foundation -F Win32_UI_WindowsAndMessaging -F Win32_Graphics_Gdi
cargo add once_cell

use windows::Win32::{
    Foundation::{HWND, LPARAM, LPARAM, RECT, WPARAM},
    Graphics::Gdi::{GetClientRect, PtInRect, ScreenToClient},
    UI::WindowsAndMessaging::*,
};
use std::collections::HashSet;

// 全局状态
static mut FORWARDING_WINDOWS: HashSet<isize> = HashSet::new();

#[tauri::command]
fn set_click_through(window: tauri::Window, enabled: bool) -> Result<(), String> {
    let hwnd = window.hwnd().unwrap();
    let hwnd = HWND(hwnd.0);
    
    unsafe {
        if enabled {
            // 设置透明窗口样式
            let ex_style = WS_EX_TRANSPARENT | WS_EX_LAYERED;
            SetWindowLongW(hwnd, GWL_EXSTYLE, ex_style.0 as i32);
            FORWARDING_WINDOWS.insert(hwnd.0);
            install_mouse_hook();
        } else {
            // 移除透明样式
            let ex_style = WS_EX_LAYERED;
            SetWindowLongW(hwnd, GWL_EXSTYLE, ex_style.0 as i32);
            FORWARDING_WINDOWS.remove(&hwnd.0);
        }
    }
    
    Ok(())
}
```

#### 前端集成

```typescript
// 根据 Canvas Alpha 通道决定是否穿透
function updateClickThrough(x: number, y: number) {
    const ctx = canvas.getContext('2d');
    const pixel = ctx.getImageData(x, y, 1, 1).data;
    
    // alpha > 128 认为是非透明区域（宠物实体）
```

## 三、Alpha Hit-Test

### 原理

通过检测 Canvas 上鼠标位置的 Alpha 通道值来决定是否响应点击。

```typescript
class AlphaHitTest {
    private canvas: HTMLCanvasElement;
    private ctx: CanvasRenderingContext2D;
    
    constructor(canvas: HTMLCanvasElement) {
        this.canvas = canvas;
        this.ctx = canvas.getContext('2d')!;
    }
    
    /**
     * 检测指定坐标是否为"实体"区域
     * @param x 相对于 Canvas 的 X 坐标
     * @param y 相对于 Canvas 的 Y 坐标
     * @param threshold Alpha 阈值 (默认 128)
     */
    isHit(x: number, y: number, threshold = 128): boolean {
        try {
            const pixel = this.ctx.getImageData(x, y, 1, 1).data;
            return pixel[3] > threshold; // alpha 通道
        } catch {
            return false;
        }
    }
    
    /**
     * 批量检测（用于拖拽）
     */
    isHitBatch(points: Array<{x: number, y: number}>, threshold = 128): boolean {
        return points.some(p => this.isHit(p.x, p.y, threshold));
    }
}
```

### 在 Vue 中使用

```typescript
// composables/useAlphaHitTest.ts
import { ref } from 'vue';

export function useAlphaHitTest(canvas: Ref<HTMLCanvasElement | null>) {
    const hitTest = ref<AlphaHitTest | null>(null);
    
    watch(canvas, (newCanvas) => {
        if (newCanvas) {
            hitTest.value = new AlphaHitTest(newCanvas);
        }
    }, { immediate: true });
    
    function handleMouseMove(e: MouseEvent) {
        if (!hitTest.value || !canvas.value) return;
        
        const rect = canvas.value.getBoundingClientRect();
        const x = e.clientX - rect.left;
        const y = e.clientY - rect.top;
        
        const isOpaque = hitTest.value.isHit(x, y);
        // 更新鼠标样式或触发穿透
        document.body.style.cursor = isOpaque ? 'pointer' : 'default';
    }
    
    return { hitTest, handleMouseMove };
}
```

## 四、综合示例：宠物实体检测

```typescript
interface PetHitArea {
    // 整体透明度检测
    checkAlphaHit(x: number, y: number): boolean;
    
    // 特定区域检测（用于精确交互）
    checkRegionHit(x: number, y: number, region: 'head' | 'body' | 'tail'): boolean;
    
    // 边界检测
    checkScreenBounds(): { onScreen: boolean; monitor: MonitorInfo | null };
}

class DesktopPetHitTest implements PetHitArea {
    private canvas: HTMLCanvasElement;
    private ctx: CanvasRenderingContext2D;
    
    // 预定义的交互区域（相对于 Canvas）
    private regions = {
        head: { x: 60, y: 20, width: 60, height: 50 },    // 头部区域
        body: { x: 40, y: 70, width: 100, height: 80 },  // 身体区域
        tail: { x: 10, y: 100, width: 30, height: 40 },  // 尾巴区域
    };
    
    constructor(canvas: HTMLCanvasElement) {
        this.canvas = canvas;
        this.ctx = canvas.getContext('2d')!;
    }
    
    checkAlphaHit(x: number, y: number): boolean {
        try {
            const pixel = this.ctx.getImageData(
                Math.floor(x), Math.floor(y), 1, 1
            ).data;
            return pixel[3] > 64; // 较低阈值，更灵敏
        } catch {
            return false;
        }
    }
    
    checkRegionHit(
        x: number, 
        y: number, 
        region: 'head' | 'body' | 'tail'
    ): boolean {
        const r = this.regions[region];
        return x >= r.x && x <= r.x + r.width
            && y >= r.y && y <= r.y + r.height
            && this.checkAlphaHit(x, y);
    }
    
    checkScreenBounds(): { onScreen: boolean; monitor: MonitorInfo | null } {
        // 调用 Rust 端获取显示器信息
        return { onScreen: true, monitor: null };
    }
}
```

## 五、已知问题与解决方案

### 1. 拖动窗口时的干扰

当拖动窗口经过设置了转发鼠标消息的区域时，可能出现位置抖动。

**解决**：使用 `SendMessage` 替代 `PostMessage`（同步 vs 异步）：

```rust
// 改为同步发送
SendMessageW(hwnd, WM_MOUSEMOVE, w, l);  // 而不是 PostMessageW
```

### 2. Hover 样式闪烁

鼠标移到具有 hover 样式的 DOM 上时可能出现闪烁。

**解决**：在转发时设置 wParam 参数：

```rust
let w = WPARAM(1);  // 而非 0
SendMessageW(hwnd, WM_MOUSEMOVE, w, l);
```

### 3. 非必要不使用

减少点击穿透的副作用：
- 窗口被完全覆盖时停用
- 切换窗口/失焦时停用
