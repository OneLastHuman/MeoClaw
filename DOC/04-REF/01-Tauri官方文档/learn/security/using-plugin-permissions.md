# 使用插件权限

_Source: https://v2.tauri.org.cn/learn/security/using-plugin-permissions/_

本练习的目标是让你更好地理解如何启用或禁用插件权限、在哪里查看权限描述，以及如何使用插件的默认权限。

学习结束后，你将能够查找和使用任意插件的权限，并理解如何定制现有权限。你将获得一个使用插件及其特定权限的 Tauri 应用示例。

  2. ### 将 `file-system` 插件添加到你的应用中

名为“将 file-system 插件添加到你的应用中”的章节

你可以通过多种资源搜索现有的插件。

最直接的方法是查看插件是否已存在于文档的[插件 (Plugins)](/plugin/) 部分，这意味着它是 Tauri 官方维护的插件集的一部分。文件系统 (Filesystem) 插件属于 Tauri 插件工作区，你可以按照[说明](/plugin/file-system/#setup)将其添加到你的项目中。

如果该插件属于社区贡献，你很可能可以通过在 [crates.io](https://crates.io/search?q=tauri-plugin-) 上搜索 `tauri-plugin-<你的插件名>` 找到它。

显示解决方案

如果是我们工作区中现有的插件，你可以使用自动化方式。

         pnpm tauri add fs

如果你在 [crates.io](https://crates.io/crates/tauri-plugin-fs) 上找到了它，则需要手动将其添加为依赖项，并修改 Tauri 构建器以初始化该插件。

终端窗口

         cargo add tauri-plugin-fs

修改 `lib.rs` 以初始化插件

src-tauri/src/lib.rs

         #[cfg_attr(mobile, tauri::mobile_entry_point)]

         fn run() {

           tauri::Builder::default()

             .plugin(tauri_plugin_fs::init())

             .run(tauri::generate_context!())

             .expect("error while running tauri application");

         }

  3. ### 了解 `fs` 插件的默认权限

名为“了解 fs 插件的默认权限”的章节

每个插件都有一个 `default`（默认）权限集，其中包含使用该插件开箱即用所需的所有权限和作用域，旨在提供最小可行功能集。

对于官方维护的插件，你可以在文档中找到渲染好的说明（例如：[fs 默认权限](/plugin/file-system/#default-permission)）。

如果你在研究社区插件，则需要查看该插件的源代码。它通常定义在 `your-plugin/permissions/default.toml` 中。

显示解决方案

         "$schema" = "schemas/schema.json"

         [default]

         description = """

         # Tauri `fs` default permissions

         This configuration file defines the default permissions granted

         to the filesystem.

         ### Granted Permissions

         This default permission set enables all read-related commands and

         allows access to the `$APP` folder and sub directories created in it.

         The location of the `$APP` folder depends on the operating system,

         where the application is run.

         In general the `$APP` folder needs to be manually created

         by the application at runtime, before accessing files or folders

         in it is possible.

         ### Denied Permissions

         This default permission set prevents access to critical components

         of the Tauri application by default.

         On Windows the webview data folder access is denied.

         """

         permissions = ["read-all", "scope-app-recursive", "deny-default"]

  4. ### 寻找正确的权限

名为“寻找正确的权限”的章节

这一步的重点是找到你需要向前端公开命令所需的权限，同时保持对系统最小的访问权限。

`fs` 插件具有自动生成的权限，可以禁用或启用单个命令，并允许或禁用全局作用域。

这些可以在[文档](/plugin/file-system/#permission-table)或插件源代码（`fs/permissions/autogenerated`）中找到。

假设我们想启用对用户 `$HOME` 文件夹下 `test.txt` 文本文件的写入功能。

为此，我们将在自动生成的权限中搜索用于启用写入文本文件的权限（如 `allow-write-text-file`），然后寻找一个允许我们访问 `$HOME/test.txt` 文件权限的作用域。

我们需要将这些内容添加到 `src-tauri/tauri.conf.json` 中的 `capabilities` 部分，或者添加到 `src-tauri/capabilities/` 文件夹中的文件中。默认情况下，`src-tauri/capabilities/default.json` 中已经存在一个可以修改的权限设置。

显示解决方案

src-tauri/capabilities/default.json

         {

           "$schema": "../gen/schemas/desktop-schema.json",

           "identifier": "default",

           "description": "Capability for the main window",

           "windows": [

             "main"

           ],

           "permissions": [

             "path:default",

             "event:default",

             "window:default",

             "app:default",

             "image:default",

             "resources:default",

             "menu:default",

             "tray:default",

             "shell:allow-open",

             "fs:default",

             "fs:allow-write-text-file",

           ]

         }

由于 `fs` 插件中只有用于访问完整 `$HOME` 文件夹的自动生成作用域，我们需要配置自己的作用域。该作用域应仅为 `write-text-file` 命令启用，并且只能公开我们的 `test.txt` 文件。

显示解决方案

src-tauri/capabilities/default.json

         {

           "$schema": "../gen/schemas/desktop-schema.json",

           "identifier": "default",

           "description": "Capability for the main window",

           "windows": [

             "main"

           ],

           "permissions": [

             "path:default",

             "event:default",

             "window:default",

             "app:default",

             "image:default",

             "resources:default",

             "menu:default",

             "tray:default",

             "shell:allow-open",

             "fs:allow-write-text-file",

             {

               "identifier": "fs:allow-write-text-file",

               "allow": [{ "path": "$HOME/test.txt" }]

             },

           ]

         }

  5. ### 在实践中测试权限

名为“在实践中测试权限”的章节

添加必要的权限后，我们要确认应用可以访问该文件并写入内容。

显示解决方案

我们可以使用此代码片段来写入文件

src/main.ts

         import { writeTextFile, BaseDirectory } from '@tauri-apps/plugin-fs';

         let greetInputEl: HTMLInputElement | null;

         async function write(message: string) {

             await writeTextFile('test.txt', message, { baseDir: BaseDirectory.Home });

         }

         window.addEventListener("DOMContentLoaded", () => {

           greetInputEl = document.querySelector("#greet-input");

           document.querySelector("#greet-form")?.addEventListener("submit", (e) => {

             e.preventDefault();

             if (!greetInputEl )

               return;

             write(greetInputEl.value == "" ? "No input provided": greetInputEl.value);

           });

         });

使用此代码片段替换 `src/main.ts`，意味着在使用纯 Vanilla+Typescript 应用时，无需修改默认的 `index.html`。在运行的应用输入框中输入任何内容，提交后都会被写入到该文件中。

现在让我们进行实践测试

         pnpm run tauri dev

在输入框中输入内容并点击“Submit”后，我们可以通过终端模拟器或手动打开主文件夹中的文件来进行验证。

         cat $HOME/test.txt

你应该能看到你的输入，并且已经完成了关于如何在 Tauri 应用中使用插件权限的学习。🥳
