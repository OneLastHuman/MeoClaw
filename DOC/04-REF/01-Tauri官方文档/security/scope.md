# 命令作用域

_Source: https://v2.tauri.org.cn/security/scope/_

作用域 (Scope) 是一种用于定义 Tauri 命令允许或禁止行为的细粒度机制。

作用域分为 `allow`（允许）或 `deny`（禁止）两种，其中 `deny` 始终优先于 `allow`。

作用域类型必须是任何支持 [`serde`](https://docs.rs/serde/latest/serde/) 序列化的类型。通常，这些类型是插件特有的。对于在 Tauri 应用程序中实现的带作用域的命令，其作用域类型需要在应用程序中定义，并在命令实现中强制执行。

例如，[`Fs`](https://github.com/tauri-apps/plugins-workspace/tree/v2/plugins/fs)（文件系统）插件允许你使用作用域来允许或拒绝访问特定的目录和文件；而 [`http`](https://github.com/tauri-apps/plugins-workspace/tree/v2/plugins/http) 插件则使用作用域来过滤允许访问的 URL。

作用域会被传递给命令，并由命令本身负责处理或强制执行。

注意

命令开发者需要确保不存在绕过作用域的可能性。应审查作用域验证的实现，以确保其正确性。

## 示例

名为“示例”的章节

以下示例取自 [`Fs`](https://github.com/tauri-apps/plugins-workspace/tree/v2/plugins/fs) 插件的权限配置：

在该插件中，所有命令的作用域类型均为字符串，其中包含一个兼容 [`glob`](https://docs.rs/glob/latest/glob/) 的路径。

plugins/fs/permissions/autogenerated/base-directories/applocaldata.toml

    [[permission]]

    identifier = "scope-applocaldata-recursive"

    description = '''

    This scope recursive access to the complete `$APPLOCALDATA` folder,

    including sub directories and files.

    '''

    [[permission.scope.allow]]

    path = "$APPLOCALDATA/**"

plugins/fs/permissions/deny-webview-data.toml

    [[permission]]

    identifier = "deny-webview-data-linux"

    description = '''

    This denies read access to the

    `$APPLOCALDATA` folder on linux as the webview data and

    configuration values are stored here.

    Allowing access can lead to sensitive information disclosure and

    should be well considered.

    '''

    platforms = ["linux"]

    [[scope.deny]]

    path = "$APPLOCALDATA/**"

    [[permission]]

    identifier = "deny-webview-data-windows"

    description = '''

    This denies read access to the

    `$APPLOCALDATA/EBWebView` folder on windows as the webview data and

    configuration values are stored here.

    Allowing access can lead to sensitive information disclosure and

    should be well considered.

    '''

    platforms = ["windows"]

    [[scope.deny]]

    path = "$APPLOCALDATA/EBWebView/**"

上述作用域可用于允许访问 `APPLOCALDATA` 文件夹，同时防止访问 Windows 上包含敏感 webview 数据的 `EBWebView` 子文件夹。

这些作用域可以合并为一个集合，这不仅减少了重复配置，还使任何查看应用程序配置的人员更容易理解。

首先，将禁止作用域合并到 `deny-default` 中：

plugins/fs/permissions/deny-default.toml

    [[set]]

    identifier = "deny-default"

    description = '''

    This denies access to dangerous Tauri relevant files and

    folders by default.

    '''

    permissions = ["deny-webview-data-linux", "deny-webview-data-windows"]

随后，将禁止和允许作用域进行合并：

    [[set]]

    identifier = "scope-applocaldata-reasonable"

    description = '''

    This scope set allows access to the `APPLOCALDATA` folder and

    subfolders except for linux,

    while it denies access to dangerous Tauri relevant files and

    folders by default on windows.

    '''

    permissions = ["scope-applocaldata-recursive", "deny-default"]

这些作用域既可以通过扩展插件的全局作用域应用于所有命令，也可以通过与权限内的已启用命令结合使用，仅应用于选定的命令。

对 `APPLOCALDATA` 中的文件进行合理的只读文件访问，配置可能如下所示：

    [[set]]

    identifier = "read-files-applocaldata"

    description = '''

    This set allows file read access to the `APPLOCALDATA` folder and

    subfolders except for linux,

    while it denies access to dangerous Tauri relevant files and

    folders by default on windows.'''

    permissions = ["scope-applocaldata-reasonable", "allow-read-file"]

这些示例仅强调了作用域功能本身。每位插件或应用程序开发者都需要根据其具体用例，考量作用域的合理组合。
