# Tauri 生态系统安全

_Source: https://v2.tauri.org.cn/security/ecosystem/_

我们的 Tauri 组织生态系统托管在 GitHub 上，并提供多项功能，使我们的存储库能够更有效地抵御针对源代码和发布版本的攻击者。

为了降低风险并遵守业界公认的最佳实践，我们采取了以下方法。

### 构建流水线

标题为“构建流水线”的章节

我们的源代码制品发布过程通过 GitHub Actions 实现高度自动化，但也强制要求由人工进行启动和审核。

### 已签名提交

标题为“已签名提交”的章节

我们的核心存储库要求对提交进行签名，以降低冒充风险，并允许在检测到潜在威胁时对归属提交进行识别。

### 代码评审

标题为“代码评审”的章节

所有合并到我们存储库的拉取请求 (PR) 都需要至少一名项目维护者的批准，在大多数情况下这指的是工作组成员。代码通常在 PR 中进行审查，并运行默认的安全工作流和检查，以确保代码符合通用标准。

### 发布流程

标题为“发布流程”的章节

我们的工作组负责审查代码变更，为 PR 标记范围，并确保一切保持最新。我们致力于在发布小版本和大版本之前，对所有涉及安全性的 PR 进行内部审计。

当需要发布新版本时，其中一位维护者会在开发分支上标记一个新版本，流程包括：

  * 验证核心组件
  * 运行测试
  * 审核 Crate 和 npm 包的安全性
  * 生成变更日志
  * 创建制品
  * 创建版本发布草稿

随后，维护者会检查发布说明，并在必要时进行编辑，最终完成新版本的发布。
