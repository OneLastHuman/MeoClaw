# Options 面板 SIZE 滑块修复

## 问题

选项面板中 SIZE 滑块的位置与显示的百分比数字不匹配。

- 滑块范围 0.5-2.0，100% 标记在 slider 中间位置
- 但 fill 进度条计算：`(1.0-0.5)/(2.0-0.5)*100 = 33%`
- 导致拖到 slider 中间（100% 标记处），fill 只显示 33%，与文字显示 100% 不一致

## 修复内容

1. **Slider 范围改为百分比直观显示**
   - 旧：`min=0.5, max=2, step=0.1`
   - 新：`min=50, max=200, step=10`

2. **新增 150% 档位**
   - 标签顺序：50% | **100%** | 150% | 200%
   - 100% 高亮显示

3. **内部转换逻辑**
   - 新增 `sliderToScale(v)`：slider 值 / 100 = 实际 scale
   - 新增 `scaleToSlider(s)`：实际 scale * 100 = slider 值
   - `applyScale()` 接收 slider 值，内部转换为 scale

4. **Fill 进度条计算修正**
   - 旧：`((value - 0.5) / (2.0 - 0.5)) * 100`
   - 新：`((value - 50) / (200 - 50)) * 100`
   - 现在 100% slider 位置 = 50% fill 进度 = 显示 100%

## 影响文件

- `src/options.ts`：滑块 HTML、JS 转换逻辑

## 验证方式

打开 Options → SIZE 面板，拖动滑块：
- 滑块在中间（100% 位置）时，fill 条应到一半
- 数字显示应为 100%
