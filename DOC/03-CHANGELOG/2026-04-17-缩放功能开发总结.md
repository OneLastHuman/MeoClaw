# 桌宠缩放功能开发会话总结

**日期**: 2026-04-17
**会话目标**: 实现桌宠大小调整功能（选项菜单滑块控制 50%-200%）

---

## 1. 用户需求

### 1.1 核心功能
- 在选项菜单中提供滑块控制，可调整桌宠在桌面的显示大小
- 滑块范围：50% - 200%，步进 10%
- 实时预览：拖动滑块时桌宠立即更新大小
- 风格参考：匹配 MeoClaw Arcade 样式（橙色主题、Space Grotesk 字体）

### 1.2 附加要求
- **Response 状态气泡缩放**：Response 状态下气泡和窗口一起缩放
- **位置保持**：切换 Response 到其他状态时，猫咪位置保持不变
- **缩放原点**：以左下角为缩放中心点
- **Working 状态**：bubble 相对猫咪的位置保持正确

## 2. 技术方案演变

### 2.1 方案一：CSS 变量逐个缩放（最终未成功）

**原理**：
- 使用 `scaledCSSVars` computed 动态计算所有元素尺寸
- CSS 变量注入到 `.pet-root`，子元素通过 `var(--xxx)` 使用
- `petScale` watcher 调用 `win.setSize()` 更新窗口物理尺寸

**失败原因**：
1. 精灵图动画计算期望基础尺寸 180px 不变
2. CSS 变量缩放后，精灵图 backgroundPosition 计算错乱
3. 内容被截断 - 窗口 DOM 尺寸没有同步跟随

### 2.2 方案二：Transform: scale()（尝试中）

**原理**：
- `.pet-root` 使用 `transform: scale(petScale)` 统一缩放
- `transform-origin: bottom left` 保证左下角为原点
- 精灵图保持 180x180 不变

**失败原因**：
1. transform 缩放只改变视觉呈现，不改变 DOM 布局尺寸
2. 内容仍被截断
3. 窗口实际大小与 CSS 尺寸不同步

### 2.3 方案三：CSS 变量 + 窗口同步（当前实现）

**原理**：
- CSS 变量控制内容尺寸：`--window-width: ${180 * petScale}px`
- 精灵图也使用 `petScale` 缩放：`width: ${180 * petScale}px`
- `petScale` watcher 调用 `win.setSize()` 同步更新窗口物理尺寸

**状态**：实现中，待测试验证

## 4. 经验教训

### 4.1 成功的经验

1. **localStorage 持久化**：`petSettings.ts` 中 `watch(petScale)` 自动保存到 localStorage
2. **CSS 变量方案**：适合 overlay 元素（输入框、气泡）的定位和尺寸缩放
3. **Tauri 事件同步**：`emit('pet-scale-changed')` + `listen()` 实现跨窗口通信
4. **petScale watcher**：`watch(petScale, ...)` 统一处理缩放时的窗口尺寸更新

### 4.2 失败的教训

1. **不要轻易删除 stores/ 目录**：导致之前实现的缩放功能完全丢失，必须从头重建

2. **精灵图缩放陷阱**：
   - 精灵图使用 `backgroundPosition` + `backgroundSize` 实现帧动画
   - 如果容器尺寸不等于 180px，必须同步缩放 `backgroundPosition` 和 `backgroundSize`
   - 计算公式：`backgroundPosition: -${bgX * petScale}px`
   - 计算公式：`backgroundSize: ${spriteW * petScale}px`

3. **transform: scale() 限制**：
   - `transform` 只改变视觉呈现，不改变 DOM 布局
   - 窗口裁剪由 DOM 布局尺寸决定，不是视觉尺寸
   - 要让内容不被截断，必须让 DOM 尺寸 = 视觉尺寸

4. **git 分支策略**：
   - 大幅修改前应创建备份分支或 commit
   - 频繁 revert 可能导致代码状态不一致

### 4.3 待解决问题

1. **Response 状态位置跳变**：切换 Response 到其他状态时，猫咪位置没有保持不变
   - 需要检查 `adjustWindowForResponse` 的计算逻辑
   - 可能与 `catPhysicalHeight` 的使用有关

2. **缩放原点**：当前实现以左上角为原点，需要验证是否以左下角为原点

3. **Working 状态 bubble 位置**：需要验证 `working-bubble-left/top` 是否正确跟随猫咪

## 8. 关键代码片段

### 8.1 petScale watcher（窗口尺寸同步）
```typescript
watch(petScale, async (newScale, oldScale) => {
  if (newScale === oldScale) return;
  const win = getCurrentWindow();
  const newWidth = Math.round(180 * newScale * 2);
  const newHeight = Math.round(180 * newScale * 2);
  await win.setSize(new PhysicalSize(newWidth, newHeight));
});
```

### 8.2 精灵图缩放计算
```typescript
const backgroundStyle = computed(() => {
  const scaledW = WINDOW_WIDTH * petScale.value;
  const scaledH = WINDOW_HEIGHT * petScale.value;
  return {
    backgroundPosition: `-${bgX * petScale.value}px -${bgY * petScale.value}px`,
    backgroundSize: `${spriteW * petScale.value}px ${spriteH * petScale.value}px`,
    width: `${scaledW}px`,
    height: `${scaledH}px`,
  };
});
```
