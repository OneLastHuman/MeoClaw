# 状态管理

## 概述

MeoClaw 采用分层的状态管理策略:
- **前端**: Vue Composition API (ref/computed)
- **后端**: 文件系统 + 内存状态
- **IPC**: Tauri 事件系统

## 前端状态管理

### Vue reactivity

使用 Vue 3 Composition API 的 `ref` 和 `computed`:

```typescript
// 单值状态
const animState = ref<AnimState>('idle');
const inputText = ref('');

// 计算属性
const showInput = computed(() =>
  !isCompactWindow.value && ['EnterInput', 'received', 'Response'].includes(animState.value)
);
```

### petSettings 状态模块

**文件**: `src/stores/petSettings.ts`

**状态**:

| 状态 | 类型 | 持久化 | 用途 |
|------|------|--------|------|
| petScale | `Ref<number>` | localStorage | 桌宠缩放比例 |
| responseSound | `Ref<ResponseSoundId>` | localStorage | 响应音效 |

**实现模式**:

```typescript
// 加载
export function loadPetScale(): number {
  return normalizePetScale(window.localStorage.getItem(PET_SCALE_STORAGE_KEY));
}

// 持久化
watch(petScale, (scale) => {
  persistPetScale(scale);
}, { flush: 'post' });
```

### DesktopPet 组件状态

**文件**: `src/components/DesktopPet.vue`

**状态分组**:

| 分组 | 状态 | 说明 |
|------|------|------|
| 动画 | animState, currentFrame, animationTimer | 精灵图动画 |
| 输入 | inputText, droppedFiles, showInput | 用户输入相关 |
| 气泡 | bubbleTools, showBubble, responseBubbleText | 工具调用/响应气泡 |
| 窗口 | windowBehaviorState, isWindowDragging | 边缘吸附行为 |
| 拖拽 | windowDragSession, dragActive | 拖拽会话 |
| 预览 | isEdgeEnterPlaying, edgeEnterFrame, edgePeekLoopFrame | 边缘动画帧 |

## 后端状态管理

### Rust 内存状态

**AppState**:

```rust
pub struct AppState {
    pub backend_manager: Arc<Mutex<BackendManager>>,
}
```

通过 `tauri::State<AppState>` 在命令间共享。

**BackendManager**:

```rust
pub struct BackendManager {
    config: Arc<Mutex<BackendConfig>>,
    clients: Arc<Mutex<HashMap<String, Box<dyn BackendClient>>>>,
}
```

### BackendClient trait

所有后端客户端必须实现此接口：

```rust
pub trait BackendClient: Send + Sync {
    fn name(&self) -> &str;
    fn check_health(&self) -> bool;
    fn send_message(&self, text: &str) -> Result<Box<dyn Stream>, Box<dyn Error + Send + Sync>>;
    fn endpoint(&self) -> &str;
    fn auth_headers(&self) -> HashMap<String, String>;
}
```

### 文件系统状态

| 文件路径 | 格式 | 用途 | 写入方 |
|----------|------|------|--------|
| `/tmp/meo_anim_state.txt` | 字符串 + 换行符 | 动画状态 | Rust |
| `/tmp/bubble_content.txt` | 字符串 | 气泡内容 | Rust |
| `~/.local/share/meo-claw/backend-config.json` | JSON | 后端配置 | Rust |

**状态文件同步流程**:

```
lib.rs (Rust) → fs::write() → FileWatcher → Frontend listen
```

## 跨进程通信

### 事件流

```
Frontend → Tauri Command → Rust 处理 → Tauri Event → Frontend
```

### 核心事件

| 事件名 | 方向 | Payload | 用途 |
|--------|------|---------|------|
| switch-animation | Rust→Frontend | `string` | 动画状态切换 |
| update-bubble | Rust→Frontend | `BubblePayload` | 工具调用气泡 |
| update-response-bubble | Rust→Frontend | `ResponseBubblePayload` | 响应气泡 |
| gateway-disconnected | Rust→Frontend | `()` | 网关断开 |
| gateway-connected | Rust→Frontend | `()` | 网关连接 |
| backend-switched | Rust→Frontend | `string` | 后端切换 |

### 命令调用

| 命令 | 参数 | 返回 | 用途 |
|------|------|------|------|
| switch_animation | state: string | () | 切换动画 |
| send_message | message: string, attachments: Attachment[] | () | 发送消息 |
| update_bubble | tools: ToolCall[] | () | 更新气泡 |
| switch_backend | backend: string | () | 切换后端 |
| check_backend_health | - | BackendHealth | 健康检查（当前选中后端） |
| check_all_backends_health | - | BackendHealth[] | 健康检查（所有已注册后端） |

### 窗口行为状态机

```typescript
type WindowBehaviorState =
  | 'normal'         // 正常显示
  | 'dragging'       // 拖拽中
  | 'edgeHiddenLeft' // 隐藏到左边缘
  | 'edgeHiddenRight' // 隐藏到右边缘
  | 'edgePeekLeft'   // 显示左边缘窥视
  | 'edgePeekRight'; // 显示右边缘窥视
```

**状态转换图**:
```
┌─────────┐  拖到边缘  ┌───────────────┐  鼠标进入  ┌───────────┐
│ normal  │ ─────────► │ edgeHidden*   │ ─────────► │ edgePeek* │
└─────────┘            └───────────────┘            └───────────┘
     ▲                        │                          │
     │◄──── 鼠标离开 ─────────┘ ◄── 延迟后 ──────────────┘
     │
     └─────────── 拖拽开始 ───────────────────────────────────┐
                                                              ▼
                                                         dragging
                                                              │
                                                       拖拽结束 ─┘
```

## 配置持久化

### 前端配置

- **存储**: localStorage
- **键**:
  - `meoclaw.petScale` (number)
  - `meoclaw.responseSound` (string)

### 后端配置

- **路径**: `~/.local/share/meo-claw/backend-config.json`
- **结构**:

```json
{
  "selected": "openclaw",
  "openclaw": {
    // endpoint 由运行时端口发现机制确定（18789 或 19001）
    "endpoint": null,
    "health_endpoint": null
  },
  "hermes": {
    "endpoint": "http://127.0.0.1:8642",
    "health_endpoint": "http://127.0.0.1:8642/health"
  },
  "discovered_instances": []
}
```
