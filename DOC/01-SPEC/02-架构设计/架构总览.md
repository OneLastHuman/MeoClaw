# 架构总览

## 项目概述

MeoClaw 是一个基于 Tauri v2 构建的跨平台桌宠应用，提供可交互的桌面宠物体验，集成了 OpenClaw 和 Hermes 两个 AI 后端。

## 技术栈

| 层次 | 技术 |
|------|------|
| 框架 | Tauri v2 |
| 前端 | Vue 3 + TypeScript + Vite |
| 后端 | Rust |
| 状态管理 | Pinia (前端) / Arc<Mutex<>> (Rust) |
| 通信 | Tauri IPC (命令 + 事件) |

## 核心模块

### 前端模块

| 模块 | 路径 | 职责 |
|------|------|------|
| DesktopPet.vue | src/components/ | 主桌宠组件，处理动画、交互、拖拽 |
| petSettings.ts | src/stores/ | 宠物配置（缩放、音效） |
| animations.ts | src/config/ | 动画状态机配置 |
| windowBehavior.ts | src/ | 边缘吸附行为逻辑 |
| BubbleWindow.vue | src/ | 气泡窗口组件 |

### 后端模块 (Rust)

| 模块 | 路径 | 职责 |
|------|------|------|
| lib.rs | src-tauri/src/ | 主入口，窗口管理，事件处理 |
| backend/mod.rs | src-tauri/src/ | 后端管理器入口 |
| backend/manager.rs | src-tauri/src/ | BackendManager 实现，多后端路由 |
| backend/client_trait.rs | src-tauri/src/ | BackendClient trait 定义 |
| backend/openclaw_client.rs | src-tauri/src/ | OpenClaw 适配器 |
| backend/hermes_client.rs | src-tauri/src/ | Hermes HTTP 客户端 |
| backend/hermes_discovery.rs | src-tauri/src/ | Hermes 服务发现 |
| openclaw/mod.rs | src-tauri/src/ | OpenClaw WebSocket 客户端 |

## 窗口模型

应用使用多窗口架构：

| 窗口 | 用途 | 配置 |
|------|------|------|
| main | 主桌宠窗口 | 180x180，可缩放 |
| bubble | 工具调用气泡 | 380x560，透明、顶层 |
| options | 设置面板 | 680x550，模态 |

## 事件流

### 动画状态切换
```
用户交互 → DesktopPet.vue → switch-animation 事件
    → lib.rs (switch_animation 命令) → 写入 /tmp/meo_anim_state.txt
    → FileWatcher 检测到变化 → emit "switch-animation" → Frontend 响应
```

### 消息发送
```
用户输入 → invoke('send_message') → BackendManager 路由
    → OpenClaw (WebSocket) 或 Hermes (HTTP/SSE)
    → 响应通过 Tauri 事件传递回前端
```

## 配置管理

- **前端配置**: localStorage (petScale, responseSound)
- **后端配置**: `~/.local/share/meo-claw/backend-config.json`
- **运行时状态**: `/tmp/meo_anim_state.txt` (动画状态), `/tmp/bubble_content.txt` (气泡内容)

## 多后端架构

BackendManager 统一管理多个 AI 后端客户端：

```
BackendManager
├── OpenClawClientAdapter (WebSocket)
└── HermesClient (HTTP/SSE)
```

通过 `BackendClient` trait 实现统一接口，支持运行时切换。
