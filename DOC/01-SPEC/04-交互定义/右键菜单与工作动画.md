# 右键菜单与工作动画

## 右键菜单

### 菜单项分类

右键菜单项分为两类：

| 分类 | 包含菜单项 | 触发方式 |
|------|-----------|---------|
| **发布菜单项** | Shock、Commands | 正常右键 |
| **调试菜单项** | Start working、Working、Working Preview、Enter Receiving、Receiving、Response、创建分身 | Shift + 右键 |

### 菜单项

| 菜单项 | 分类 | 行为 |
|--------|------|------|
| Shock | 发布 | 发送 "FASTER!" 消息给 OpenClaw，播放 shock 动画（10帧单次播放） |
| Start working | 调试 | 切换到 `startworking` 动画 |
| Working | 调试 | 切换到 `working` 动画 |
| Working Preview | 调试 | 切换到 `workingPreview` 动画 |
| Enter Receiving | 调试 | 切换到 `EnterReceiving` 动画 |
| Receiving | 调试 | 切换到 `Receiving` 动画 |
| Response | 调试 | 切换到 `Response` 动画 |
| 创建分身 | 调试 | 预留功能（暂无实现） |
| Commands → /new | 发布 | 发送 `/new` 命令给 OpenClaw |
| Commands → /stop | 发布 | 发送 `/stop` 命令给 OpenClaw |
| Commands → /status | 发布 | 发送 `/status` 命令给 OpenClaw |

### 实现

右键菜单使用 Tauri v2 的 `Menu`、`MenuItem`、`Submenu` API 构建。

**菜单分类逻辑**（`src/menu/contextMenu.ts`）：

```typescript
// showAll: 是否显示调试用的动画切换菜单项
export async function showContextMenu(showAll = false) {
  // ... 创建所有菜单项 ...

  const debugItems = showAll
    ? [startWorkingItem, workingItem, workingPreviewItem,
       enterReceivingItem, receivingItem, responseItem, createCloneItem]
    : [];

  const menu = await Menu.new({
    items: [shockItem, ...debugItems, commandsSubmenu],
  });

  const win = getCurrentWindow();
  await menu.popup(undefined, win);
}
```

**触发方式**（`src/components/DesktopPet.vue`）：

```typescript
function onContextMenu(event: MouseEvent) {
  event.preventDefault();
  void showContextMenu(event.shiftKey);  // Shift+右键显示调试项
}
```

正常右键时 `showAll=false`，仅显示发布菜单项（Shock、Commands）。
按住 Shift 键同时右键时 `showAll=true`，显示全部菜单项（含调试项）。

菜单项点击后通过回调函数 `switchAnimCallback` 通知 `DesktopPet.vue` 切换动画状态。

### Shock 菜单特殊处理

Shock 菜单项点击时，会循环使用三条消息（每点击一次切换）：
1. "FASTER!"
2. "FASTER! FASTER!"
3. "FASTER! FASTER! FASTER!"

```typescript
const fasterMessages = ['FASTER!', 'FASTER! FASTER!', 'FASTER! FASTER! FASTER!'];
const message = fasterMessages[shockClickCount % 3];
shockClickCount++;
await invoke('send_to_openclaw', { message, attachments: [] });
switchAnimCallback?.('shock');
```

## 工作动画（Working）

### Start working → Working 流程

1. **Start working** (`startworking`): 播放帧0~13（14帧），一次播放，完成后自动切换到 `working`
2. **Working** (`working`): 播放帧13~47（35帧），正向循环播放

两个动画共用同一张精灵图 `startworking.png`（7×7排列），通过 `startFrame` 偏移实现分段播放。

```typescript
startworking: {
  cols: 7,
  rows: 7,
  totalFrames: 14,
  interval: 100,
  loop: false,
  startFrame: 0,
  imagePath: '/anim/startworking.png',
  nextState: 'working',  // 播完后自动切换
},
working: {
  cols: 7,
  rows: 7,
  totalFrames: 35,
  interval: 100,
  loop: true,
  pingpong: false,
  startFrame: 13,  // 从第13帧开始
  imagePath: '/anim/startworking.png',
},
```

### 非循环动画自动切换逻辑

```typescript
// 动画播放完一次的检测
const relativeFrame = currentFrame.value - firstFrame;
const nextRelative = (relativeFrame + 1) % totalFrames.value;

if (nextRelative === 0) {
  stopAnimation();
  if (anim.nextState) {
    switchAnim(anim.nextState);
  }
  return;
}
```

### Working Preview

`workingPreview` 与 `working` 使用相同的精灵图和帧范围，但用途不同：
- `working`: 实际工作时播放
- `workingPreview`: 预览模式，显示 "preview mode" 在气泡中
