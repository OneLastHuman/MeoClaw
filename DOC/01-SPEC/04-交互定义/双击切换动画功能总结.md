# 双击切换动画

## 功能描述

在桌宠窗口上双击（两次点击间隔 < 300ms），在 `idle` / `working` 与 `EnterInput` 动画之间切换。

- `idle` 或 `working` → 双击 → 切换到 `EnterInput`
- `EnterInput` → 双击 → 切换到 `idle`
- 其他状态 → 双击 → 切换到 `idle`

## 双击检测

使用自制的 mousedown 双击检测，不使用原生 `@dblclick` 事件。原因：原生 `dblclick` 在两次 `mousedown` 之后触发，与自制双击检测同时存在会导致一次双击触发两次切换。

```typescript
let lastClickTime = 0;

async function onPointerDown(event: PointerEvent) {
  if (event.button !== 0) return;

  const now = Date.now();
  // ...

  if (now - lastClickTime < 300) {
    // 双击，切换动画
    if (animState.value === 'idle' || animState.value === 'working') {
      if (cancelFromEnterInput && animState.value === 'working') {
        cancelFromEnterInput = false;
      } else {
        switchAnim('EnterInput');
      }
    } else {
      switchAnim('idle');
    }
  }
  // ...
  lastClickTime = now;
}
```

`cancelFromEnterInput` 标志用于处理以下场景：用户从 `EnterInput` 状态点击输入框外部（触发 blur 回调会切换到 `idle`），然后快速双击，此时不应该切换到 `EnterInput` 而是保持 `idle`。

## 单击 vs 双击的区分

300ms 内第二次点击 → 双击，切换动画；300ms 外第二次点击 → 单击，开始拖拽窗口。

```
第1次点击 → 记录 lastClickTime
第2次点击:
  间隔 < 300ms → 双击，切换动画
  间隔 >= 300ms → 单击，拖拽窗口
```

## 拖拽区域

只有点击在猫身区域（`catWidth × catHeight`）内才触发双击检测和拖拽：

```typescript
const clickX = event.offsetX;
const clickY = event.offsetY;
const dragWidth = scaledCSSVars.value.catWidth;
const dragHeight = scaledCSSVars.value.catHeight;
const isInDragArea = clickX >= 0 && clickX <= dragWidth && clickY >= 0 && clickY <= dragHeight;
```

## 从隐藏/peek 状态进入

从 `edgeHiddenLeft`、`edgeHiddenRight`、`edgePeekLeft`、`edgePeekRight` 状态开始拖拽时，需要先取消边缘动画，然后开始拖拽：

```typescript
if ((isEdgeEnterPlaying.value || isEdgePeekLoopPlaying.value) && (isWindowHidden.value || isWindowPeek.value)) {
  cancelEdgeEnterAnimation();
  cancelEdgePeekLoopAnimation();
  cancelPeekLeaveTimer();
}
```
