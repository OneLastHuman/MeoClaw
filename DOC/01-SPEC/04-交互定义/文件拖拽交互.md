# 文件拖拽交互

## 功能概述

用户拖动文件到桌宠窗口上方时，触发文件接收动画；松开后显示拖入文件的图标列表，支持分类展示和单独移除。发送消息时，拖放的文件路径会附加在消息文本中发送给 OpenClaw。

## 交互流程

### 状态切换

| 阶段 | 触发条件 | 动画状态 | 说明 |
|------|----------|----------|------|
| 进入 | 拖动文件到窗口 | `EnterReceiving` → `Receiving` | 先播放一次入场动画（帧0~5），然后循环等待（帧6~11） |
| 接收 | 在窗口内松开文件 | `received` | 显示文件图标 + 输入框 |
| 取消 | 在窗口外松开鼠标 | 恢复原状态 | 若 `droppedFiles` 为空则退回 `idle`，否则保持 `received` |
| 移除 | 点击文件图标的关闭按钮 | 保持在 `received` | 移除该文件，后续文件填补进来 |
| 发送 | 输入文字后按 Enter | `working` | 文字+文件路径发送给 OpenClaw，文件列表清空 |

### Tauri 拖拽事件监听

使用 `getCurrentWindow().onDragDropEvent()` 监听：

```typescript
async function setupDragDrop() {
  const win = getCurrentWindow();
  return win.onDragDropEvent((event) => {
    const payload = event.payload;

    if (payload.type === 'enter') {
      dragActive = true;
      switchAnim('EnterReceiving');
      return;
    }

    if (payload.type === 'drop') {
      dragActive = false;
      const paths: string[] = payload.paths ?? [];
      const newFiles: DroppedFile[] = paths.map((path) => {
        const name = path.split('/').pop() ?? path;
        return { name, icon: getFileIcon(name), path };
      });
      // 去重后追加
      const existingNames = new Set(droppedFiles.value.map((file) => file.name));
      const uniqueNewFiles = newFiles.filter((file) => !existingNames.has(file.name));
      droppedFiles.value = [...droppedFiles.value, ...uniqueNewFiles];
      switchAnim('received');
      return;
    }

    if (payload.type === 'leave') {
      dragActive = false;
      if (droppedFiles.value.length === 0) {
        switchAnim('idle');
      } else {
        switchAnim('received');
      }
    }
  });
}
```

## 文件图标展示

### 展示规则

- 最多并列显示 **3个** 文件图标
- 图标横向居中排列
- 图标保持原始比例，最大宽高 **40px**
- 图标下方显示截断后的文件名（超过10字符截断，保留扩展名）
- 每个图标右上角有关闭按钮（×）

### 文件自动填补逻辑

1. 拖入 N 个文件（N > 3），只显示前3个
2. 用户关闭任意一个 → 已显示的文件左侧不动，右侧未显示的文件填补进来
3. 始终保持最多3个显示，新填补的文件靠右

**示例：**
- 拖入 5 个文件 → 显示：文件1、文件2、文件3
- 关闭文件2 → 显示：文件1、文件3、文件4
- 关闭文件1 → 显示：文件3、文件4、文件5

## OpenClaw 文件路径发送

### 消息格式

文件路径作为文本附加在消息中，OpenClaw 使用 `read` 工具读取文件内容。

**单文件：**
```
用户输入的问题文字

[附件]
文件: /Users/hue/Desktop/xiaogushi.rtfd
类型: application/octet-stream
```

**多文件：**
```
用户输入的问题文字

[附件]
文件: /Users/hue/Desktop/file1.txt
类型: text/plain
文件: /Users/hue/Desktop/file2.png
类型: image/png
```

### 消息文本构建

```typescript
let fullMessage = text;
if (droppedFiles.value.length > 0) {
  const attachLines = ['\n\n[附件]'];
  for (const file of droppedFiles.value) {
    const ext = file.name.split('.').pop()?.toLowerCase() ?? '';
    attachLines.push(`文件: ${file.path}`);
    attachLines.push(`类型: ${getMimeType(ext)}`);
  }
  fullMessage += attachLines.join('\n');
}
await invoke('send_message', { message: fullMessage, attachments: [] });
```
