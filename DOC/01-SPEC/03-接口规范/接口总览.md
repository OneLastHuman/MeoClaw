# 接口总览

本文档描述 MeoClaw 桌宠应用中 Rust 后端与 Frontend 之间的所有通信接口。

## 通信架构

```
┌─────────────────────────────────────────────────────────────┐
│                      Frontend (Vue)                         │
│  DesktopPet.vue  │  options.ts  │  contextMenu.ts          │
└──────────┬────────────────┬──────────────────┬─────────────┘
           │                │                  │
           │  invoke()       │  invoke()        │  invoke()
           │  listen()       │  listen()        │
           ▼                 ▼                  ▼
┌─────────────────────────────────────────────────────────────┐
│                    Rust Backend (Tauri)                     │
│  #[tauri::command]  │  emit() 事件  │  窗口管理              │
└─────────────────────────────────────────────────────────────┘
```

## 接口类型

### 1. 命令调用 (Command Invocation)

Frontend → Rust 的同步/异步函数调用。

| 命令名称 | 调用来源 | 功能 |
|---------|---------|------|
| `switch_animation` | DesktopPet.vue | 切换桌宠动画状态 |
| `send_to_openclaw` | contextMenu.ts | 发送消息到 OpenClaw |
| `send_to_hermes` | DesktopPet.vue | 发送消息到 Hermes |
| `send_message` | DesktopPet.vue | 统一消息路由（自动选择后端） |
| `update_bubble` | 测试/内部 | 更新工具气泡 |
| `show_bubble_window` | DesktopPet.vue | 显示气泡窗口 |
| `hide_bubble_window` | DesktopPet.vue | 隐藏气泡窗口 |
| `update_bubble_content` | 内部 | 更新气泡内容 |
| `move_bubble_window` | DesktopPet.vue | 移动气泡窗口位置 |
| `enter_response_mode` | DesktopPet.vue | 进入响应模式（扩展窗口） |
| `exit_response_mode` | DesktopPet.vue | 退出响应模式（恢复窗口） |
| `switch_backend` | options.ts | 切换后端 |
| `get_current_backend` | options.ts | 获取当前后端 |
| `check_backend_health` | options.ts | 检测当前后端健康状态 |
| `check_all_backends_health` | options.ts | 检测所有后端健康状态 |
| `test_openclaw` | 测试 | 测试 OpenClaw 连接 |
| `test_bubble` | 测试 | 测试气泡事件 |
| `test_response_bubble` | 测试 | 测试响应气泡 |
| `test_hermes` | 测试 | 测试 Hermes 连接 |

### 2. 事件系统 (Event System)

Rust → Frontend 的异步事件推送。

#### 动画与状态事件

| 事件名称 | 触发来源 | 用途 |
|---------|---------|------|
| `switch-animation` | 动画切换/文件监控 | 通知前端切换动画状态 |
| `update-bubble` | 工具调用回调 | 更新工具执行气泡 |
| `update-response-bubble` | 响应回调 | 显示最终响应文本 |
| `backend-switched` | 后端切换 | 通知前端后端已切换 |

#### 网关连接事件

| 事件名称 | 触发来源 | 用途 |
|---------|---------|------|
| `gateway-disconnected` | WebSocket断开/HTTP轮询 | 通知前端网关断开 |
| `gateway-connected` | WebSocket重连 | 通知前端网关已连接 |
| `gateway-reconnected` | HTTP轮询恢复 | 通知前端网关恢复 |

### 3. 窗口通信

| 窗口名称 | 类型 | 说明 |
|---------|------|------|
| `main` | 主窗口 | 桌宠主窗口 (180x180 / 1120x1480) |
| `bubble` | 子窗口 | 工具气泡窗口 (380x560) |
| `options` | 子窗口 | 选项配置窗口 (680x550) |

### 4. 后端系统

| 后端 | 协议 | 用途 |
|-----|------|------|
| OpenClaw | WebSocket | 主要 AI 对话后端 |
| Hermes | HTTP (SSE) | 备用/辅助 AI 对话后端 |

## 数据结构

### AnimState (动画状态枚举)

```rust
pub enum AnimState {
    Idle,           // 空闲状态
    Shock,          // 惊讶状态
    EnterInput,     // 进入输入
    StartWorking,   // 开始工作
    Working,        //工作中
    EnterReceiving, // 进入接收
    Receiving,      // 接收中
    Received,       // 已接收
    Response,       // 响应中
}
```

### ToolCall (工具调用结构)

```rust
struct ToolCall {
    name: String,    // 工具名称
    status: String,  // 执行状态 (running/completed)
    detail: String,  // 工具详情
}
```

### BubblePayload (气泡载荷)

```rust
struct BubblePayload {
    tools: Vec<ToolCall>,
}
```

### ResponseBubblePayload (响应气泡载荷)

```rust
struct ResponseBubblePayload {
    text: String,  // 响应文本
}
```

### BackendHealth (后端健康状态)

```rust
struct BackendHealth {
    available: bool,      // 是否可用
    backend: String,      // 后端名称
    endpoint: String,     // 端点地址
    error: Option<String>, // 错误信息
}
```

## 文件监控

系统通过监控 `/tmp/meo_anim_state.txt` 文件实现状态同步：

- Rust 端写入状态
- 文件监控线程检测变化并 emit `switch-animation` 事件
- Frontend 监听事件并更新 UI
