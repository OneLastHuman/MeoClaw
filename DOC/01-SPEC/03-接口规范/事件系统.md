# 事件系统

本文档描述 MeoClaw 应用中的所有事件定义及用途。

## 事件概览

### Rust → Frontend 事件

| 事件名称 | 载荷类型 | 触发来源 |
|---------|---------|---------|
| `switch-animation` | `String` | 动画切换、文件监控 |
| `update-bubble` | `BubblePayload` | 工具调用回调 |
| `update-response-bubble` | `ResponseBubblePayload` | 响应回调 |
| `bubble-update` | `serde_json::Value` | 气泡内容更新 |
| `backend-switched` | `String` | 后端切换完成 |
| `gateway-disconnected` | `()` | WebSocket/HTTP 断开 |
| `gateway-connected` | `()` | WebSocket 重连 |
| `gateway-reconnected` | `()` | HTTP 轮询恢复 |

### Frontend → Rust 事件

| 事件名称 | 载荷类型 | 触发来源 |
|---------|---------|---------|
| `pet-scale-changed` | `{ scale: number }` | 缩放比例变更 |
| `response-sound-changed` | `{ soundId: string }` | 响应音效变更 |

## 事件详情

### switch-animation

**用途**: 通知前端切换桌宠动画状态

**触发来源**:
1. `switch_animation` 命令 (lib.rs:195)
2. 文件监控线程 (lib.rs:967)
3. OpenClaw 生命周期回调 (lib.rs:814)
4. Hermes 流处理结束 (lib.rs:323, 391)

**前端监听** (`DesktopPet.vue:1417`):
```typescript
unlistenSwitch = await listen<string>('switch-animation', (event) => {
    const state = event.payload as AnimState;
    if (state && ANIMATIONS[state]) {
        switchAnim(state);
    }
});
```

**有效载荷**:
```
idle | shock | EnterInput | startworking | working | workingPreview
| EnterReceiving | Receiving | received | Response
```

**状态转换流程**:
```
用户输入 → startworking → OpenClaw处理 → [工具调用] → update-bubble
                                                        ↓
                                                    [工具结束]
                                                        ↓
Response ← update-response-bubble
```

---

### update-bubble

**用途**: 更新工具执行气泡，显示当前正在执行的工具

**触发来源**:
1. `update_bubble` 命令 (lib.rs:517)
2. OpenClaw 工具回调 (lib.rs:840)
3. Hermes 思考中状态 (lib.rs:279)

**前端监听** (`DesktopPet.vue:1424`):
```typescript
unlistenBubble = await listen<BubblePayload>('update-bubble', (event) => {
    if (event.payload.tools) {
        bubbleTools.value = event.payload.tools;
    }
});
```

**载荷结构**:
```typescript
interface BubblePayload {
    tools: ToolCall[];
}

interface ToolCall {
    name: string;    // 工具名称
    status: string;  // "start" | "update" | "end"
    detail: string;  // 工具详情（文件名、URL等）
}
```

**工具名称映射**:
| 工具名 | detail 来源 |
|-------|------------|
| `read`, `read_dir` | `path` 字段（截取文件名） |
| `write`, `edit`, `apply_patch` | `path` 字段 |
| `exec`, `bash` | `command`/`cmd`/`script` 字段 |
| `browser` | `url` 或 `action` 字段 |
| `web_search` | `query` 字段 |
| `sessions_spawn`, `subagents` | `agent_id` 或 `name` 字段 |

---

### update-response-bubble

**用途**: 显示 AI 最终响应文本气泡

**触发来源**:
1. `test_response_bubble` 命令 (lib.rs:453)
2. OpenClaw 响应回调 (lib.rs:865)
3. Hermes 流处理结束 (lib.rs:322, 389)
4. 气泡文件监控 (lib.rs:1019)

**前端监听** (`DesktopPet.vue:1431`):
```typescript
unlistenResponseBubble = await listen<ResponseBubblePayload>(
    'update-response-bubble',
    (event) => {
        if (event.payload.text && event.payload.text !== lastResponseText) {
            lastResponseText = event.payload.text;
            responseBubbleText.value = event.payload.text;
            playResponseSound();
        }
    },
);
```

**载荷结构**:
```typescript
interface ResponseBubblePayload {
    text: string;  // 响应文本（支持 Markdown）
}
```

---

### bubble-update

**用途**: 更新气泡内容（与 `update-bubble` 不同的事件）

**触发来源**:
1. `update_bubble_content` 命令 (lib.rs:577)

**注意**: 此事件与其他气泡事件不同，载荷是原始 JSON，格式为 `{ "text": string }`。

---

### backend-switched

**用途**: 通知前端后端切换已完成

**触发来源**:
1. 托盘菜单切换到 OpenClaw (lib.rs:1118)
2. 托盘菜单切换到 Hermes (lib.rs:1128)

**前端监听** (`options.ts:934`):
```typescript
listen('backend-switched', (event: any) => {
    // event.payload 为 "openclaw" 或 "hermes"
});
```

**载荷**: `"openclaw"` 或 `"hermes"`

---

### gateway-disconnected

**用途**: 通知前端网关连接断开

**触发来源**:
1. OpenClaw WebSocket 断开 (lib.rs:894)
2. HTTP 轮询检测到断开 (lib.rs:1214)
3. OpenClaw 客户端错误 (lib.rs:918, 929)

**载荷**: 空 `()`

**前端处理**: 显示断开连接状态提示

---

### gateway-connected

**用途**: 通知前端网关连接已恢复

**触发来源**:
1. OpenClaw WebSocket 重连 (lib.rs:900)

**载荷**: 空 `()`

---

### gateway-reconnected

**用途**: 通知前端通过 HTTP 轮询恢复连接

**触发来源**:
1. HTTP 轮询首次连接成功 (lib.rs:1198)

**载荷**: 空 `()`

## 事件流程图

### 消息发送流程 (OpenClaw)

```
send_message()
    ↓
sessions_send() → WebSocket → OpenClaw Gateway
    ↓
[Lifecycle: start]
    ↓ emit: switch-animation("working")
    ↓
[Tool: start/update]
    ↓ emit: update-bubble(ToolCall)
    ↓
[Tool: end]
    ↓ emit: update-bubble(empty)
    ↓
[Response]
    ↓ emit: update-response-bubble(text)
    ↓ emit: switch-animation("Response")
```

### 消息发送流程 (Hermes)

```
send_message() [Hermes 后端]
    ↓
HTTP POST /v1/chat/completions
    ↓
[SSE stream]
    ↓ emit: update-bubble("思考中...")
    ↓
[DONE]
    ↓ emit: update-bubble(empty)
    ↓ emit: update-response-bubble(text)
    ↓ emit: switch-animation("Response")
```

### 后端切换流程

```
switch_backend("hermes")
    ↓
BackendManager.switch()
    ↓ emit: backend-switched("hermes")
```

## 文件监控与事件

系统通过监控 `/tmp/meo_anim_state.txt` 实现状态同步：

1. **写入源**:
   - OpenClaw 生命周期回调
   - Hermes 流处理结束
   - `switch_animation` 命令

2. **监控线程** (`lib.rs:935`):
   ```rust
   std::thread::spawn(move || {
       // 监控 STATE_FILE
       // 检测到修改 → emit("switch-animation", state)
   });
   ```

3. **状态值**:
   ```
   idle | working | Response
   ```

## 托盘菜单事件

托盘菜单操作触发的事件：

| 菜单项 | 触发事件 |
|-------|---------|
| 选择后端 → OpenClaw | `backend-switched("openclaw")` |
| 选择后端 → Hermes | `backend-switched("hermes")` |
| (内部) WebSocket 断开 | `gateway-disconnected()` |
| (内部) WebSocket 重连 | `gateway-connected()` |
| (内部) HTTP 轮询恢复 | `gateway-reconnected()` |
